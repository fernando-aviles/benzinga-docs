<div id="overview">
  ## 概要
</div>

Benzinga TypeScript/JavaScript SDK は、Benzinga API と連携するための、モジュール型のイベントベースインターフェースを提供します。TypeScript で記述されたこの SDK は、ブラウザ環境と Node.js 環境の両方でシームレスに動作し、キャッシュ、ディープ比較、そのほかの高度な機能を備えた拡張実装を提供します。

<div id="key-features">
  ## 主な機能
</div>

* **TypeScript サポート** - TypeScript の型定義とインターフェースを完全にサポート
* **ユニバーサル** - ブラウザ環境と Node.js 環境の両方で動作
* **モジュラーアーキテクチャ** - 必要なモジュールだけをインストール可能
* **イベントベース** - リアルタイムデータ向けのリアクティブプログラミングパターンに対応
* **高度な機能** - キャッシュ、ディープ比較、最適化を標準搭載
* **モダン** - async/await をサポートする ES6+ 構文

<div id="requirements">
  ## 要件
</div>

* **Node.js 14 以降**

<div id="installation">
  ## インストール
</div>

SDK はモジュール構成を採用しています。まず、コアの Session モジュールをインストールしてください。

```bash
npm install @benzinga/session
```

必要に応じて、追加のモジュールをインストールします。

```bash
# 例: 特定のモジュールをインストールする
npm install @benzinga/calendar-data
npm install @benzinga/quotes
npm install @benzinga/news-data
```

<div id="getting-started">
  ## はじめに
</div>

<div id="session-setup">
  ### Session のセットアップ
</div>

`@benzinga/session` モジュールは、Benzinga API で認証するための基盤となるものです。その他のすべてのモジュールは、この Session オブジェクトに依存します。

```typescript
import { Session } from '@benzinga/session';

// APIキーでSessionを初期化する
const session = new Session({
  apiKey: 'YOUR_API_KEY'
});
```

<div id="configuration-options">
  ### 設定オプション
</div>

Session オブジェクトでは、動作をカスタマイズするためのさまざまな設定オプションを利用できます。

```typescript
const session = new Session({
  apiKey: 'YOUR_API_KEY',
  environment: 'production', // または 'sandbox'
  timeout: 30000,            // リクエストタイムアウト（ミリ秒）
  // 追加の設定オプション
});
```

<div id="core-concepts">
  ## 基本概念
</div>

<div id="modular-design">
  ### モジュール設計
</div>

各 Benzinga API ドメインは、それぞれ独立した npm モジュールとしてパッケージ化されています。これにより、次のことが可能になります。

* 必要なものだけをインストールする
* バンドルサイズを削減する
* 責務を明確に分離する
* モジュールを個別に更新する

<div id="event-based-architecture">
  ### イベントベースのアーキテクチャ
</div>

このSDKでは、リアルタイムのデータストリームや更新の処理に、イベント駆動型の方式を採用しています。

```typescript
// 例: データ更新のサブスクライブ
dataManager.subscribe((data) => {
  console.log('New data received:', data);
});

// 例: イベントの処理
dataManager.on('update', (event) => {
  console.log('Data updated:', event);
});
```

<div id="caching-performance">
  ### キャッシュとパフォーマンス
</div>

このSDKには、次の目的で組み込みのキャッシュ機構が用意されています。

* 不要なAPI呼び出しを削減する
* レスポンス時間を短縮する
* 帯域幅の使用を最適化する
* オフライン時のフォールバックを提供する

<div id="deep-comparison">
  ### ディープ比較
</div>

高度なデータ比較機能により、次のことが可能になります。

* ネストされたオブジェクト内の変更の検出
* 効率的な状態管理
* 適切な更新トリガー
* UIアプリケーションにおける再レンダリングの削減

<div id="typescript-support">
  ## TypeScript サポート
</div>

SDK は TypeScript で記述されており、完全な型定義が提供されています。

```typescript
import { Session } from '@benzinga/session';
import { CalendarData, DividendEvent } from '@benzinga/calendar-data';

// TypeScript はオートコンプリートと型チェックを提供します
const session = new Session({ apiKey: 'YOUR_API_KEY' });
const calendar = new CalendarData(session);

// 型安全なAPI呼び出し
const dividends: DividendEvent[] = await calendar.getDividends({
  dateFrom: '2024-01-01',
  dateTo: '2024-12-31',
  ticker: 'AAPL'
});
```

<div id="real-time-data-streams">
  ### リアルタイムデータストリーム
</div>

```typescript
import { Session } from '@benzinga/session';
import { QuoteStream } from '@benzinga/quotes';

const session = new Session({ apiKey: 'YOUR_API_KEY' });
const quoteStream = new QuoteStream(session);

// リアルタイム気配値を購読する
quoteStream.subscribe(['AAPL', 'MSFT', 'GOOGL'], (quote) => {
  console.log('Quote update:', quote);
});

// 完了したら購読を解除する
quoteStream.unsubscribe(['AAPL']);
```

<div id="error-handling">
  ## エラー処理
</div>

try-catch ブロックを使用して、エラーを適切に処理します。

```typescript
import { Session } from '@benzinga/session';
import { NewsData } from '@benzinga/news-data';

const session = new Session({ apiKey: 'YOUR_API_KEY' });
const news = new NewsData(session);

try {
  const articles = await news.getNews({ pageSize: 10 });
  console.log(articles);
} catch (error) {
  if (error.code === 'UNAUTHORIZED') {
    console.error('Invalid API key');
  } else if (error.code === 'RATE_LIMIT') {
    console.error('Rate limit exceeded');
  } else {
    console.error('An error occurred:', error.message);
  }
}
```

<div id="pagination">
  ## ページネーション
</div>

ページ分割された結果を効率的に処理します：

```typescript
import { Session } from '@benzinga/session';
import { NewsData } from '@benzinga/news-data';

const session = new Session({ apiKey: 'YOUR_API_KEY' });
const news = new NewsData(session);

// 複数ページを取得
let page = 0;
let allArticles = [];

while (page < 5) {
  const articles = await news.getNews({
    page: page,
    pageSize: 100
  });

  if (articles.length === 0) break;

  allArticles = allArticles.concat(articles);
  page++;
}

console.log(`Fetched ${allArticles.length} articles`);
```

<div id="caching-strategy">
  ## キャッシュ戦略
</div>

パフォーマンス向上のため、組み込みキャッシュを活用します。

```typescript
import { Session } from '@benzinga/session';
import { CalendarData } from '@benzinga/calendar-data';

const session = new Session({
  apiKey: 'YOUR_API_KEY',
  cache: {
    enabled: true,
    ttl: 300000  // 5分間キャッシュ
  }
});

const calendar = new CalendarData(session);

// 最初の呼び出しはAPIにアクセス
const earnings1 = await calendar.getEarnings({ ticker: 'AAPL' });

// 2回目の呼び出しはキャッシュされたデータを使用（TTL以内の場合）
const earnings2 = await calendar.getEarnings({ ticker: 'AAPL' });
```

<div id="best-practices">
  ## ベストプラクティス
</div>

<div id="1-reuse-session-objects">
  ### 1. Session オブジェクトを再利用する
</div>

Session インスタンスを 1 つ作成し、アプリケーション全体で使い回します。

```typescript
// session.ts
export const globalSession = new Session({ apiKey: process.env.BENZINGA_API_KEY });

// 他のファイルで使用する場合
import { globalSession } from './session';
const news = new NewsData(globalSession);
```

<div id="2-environment-variables">
  ### 2. 環境変数
</div>

APIキーは環境変数に安全に格納します。

```typescript
// .envファイル
BENZINGA_API_KEY=your_api_key_here

// コード内
const session = new Session({
  apiKey: process.env.BENZINGA_API_KEY
});
```

<div id="3-type-safety">
  ### 3. 型安全性
</div>

型安全な API 操作には TypeScript を活用してください。

```typescript
import type { NewsArticle, NewsParams } from '@benzinga/news-data';

const params: NewsParams = {
  pageSize: 10,
  displayOutput: 'full',
  ticker: 'AAPL'
};

const articles: NewsArticle[] = await news.getNews(params);
```

<div id="4-error-boundaries">
  ### 4. エラーバウンダリ
</div>

本番環境では、エラーバウンダリを実装してください。

```typescript
const safeApiCall = async (fn: () => Promise<any>) => {
  try {
    return await fn();
  } catch (error) {
    console.error('API Error:', error);
    // エラートラッキングサービスにログを記録
    return null;
  }
};

const articles = await safeApiCall(() => news.getNews({ pageSize: 10 }));
```
