<div id="overview">
  ## 概要
</div>

Benzinga TypeScript/JavaScript SDK は、Benzinga APIs と連携するための、モジュール型のイベントベースインターフェースを提供します。TypeScript で記述されたこの SDK は、ブラウザ環境と Node.js 環境の両方でシームレスに動作し、キャッシュやディープ比較などの高度な機能を備えています。

<div id="key-features">
  ## 主な機能
</div>

* **TypeScript Support** - TypeScript の型定義とインターフェースを完全にサポート
* **Universal** - ブラウザ環境と Node.js 環境の両方で動作
* **Modular Architecture** - 必要なモジュールだけをインストール可能
* **Event-Based** - リアルタイムデータ向けのリアクティブなプログラミングパターン
* **Advanced Features** - キャッシュ、ディープ比較、最適化機能を標準搭載
* **Modern** - async/await をサポートした ES6+ 構文

<div id="requirements">
  ## 要件
</div>

* **Node.js 14 以降**

<div id="installation">
  ## インストール
</div>

SDK はモジュール構成を採用しています。まず、コアセッションモジュールをインストールします。

```bash
npm install @benzinga/session
```

次に、必要に応じて追加のモジュールをインストールします。

```bash
# 例: 特定のモジュールをインストールする
npm install @benzinga/calendar-data
npm install @benzinga/quotes
npm install @benzinga/news-data
```

<div id="getting-started">
  ## はじめに
</div>

<div id="session-setup">
  ### Sessionのセットアップ
</div>

`@benzinga/session` モジュールは、Benzinga APIs で認証を行うための基盤となるモジュールです。ほかのすべてのモジュールは、このSessionオブジェクトに依存します。

```typescript
import { Session } from '@benzinga/session';

// APIキーでセッションを初期化する
const session = new Session({
  apiKey: 'YOUR_API_KEY'
});
```

<div id="configuration-options">
  ### 設定オプション
</div>

Session オブジェクトでは、動作をカスタマイズするためのさまざまな設定オプションを使用できます。

```typescript
const session = new Session({
  apiKey: 'YOUR_API_KEY',
  environment: 'production', // または 'sandbox'
  timeout: 30000,            // リクエストタイムアウト（ミリ秒）
  // 追加の設定オプション
});
```

<div id="core-concepts">
  ## 基本概念
</div>

<div id="modular-design">
  ### モジュール型設計
</div>

各 Benzinga API ドメインは、個別の npm モジュールとしてパッケージ化されています。これにより、次のことが可能です。

* 必要なものだけをインストールする
* バンドルサイズを削減する
* 関心の分離を明確に保つ
* モジュールを個別に更新する

<div id="event-based-architecture">
  ### イベントベースのアーキテクチャ
</div>

SDKは、リアルタイムのデータストリームや更新を処理するために、イベント駆動型のパターンを採用しています。

```typescript
// 例: データ更新のサブスクライブ
dataManager.subscribe((data) => {
  console.log('New data received:', data);
});

// 例: イベントの処理
dataManager.on('update', (event) => {
  console.log('Data updated:', event);
});
```

<div id="caching-performance">
  ### キャッシュとパフォーマンス
</div>

SDK には、次のことを実現する組み込みのキャッシュ機能が含まれています。

* 不要な API 呼び出しを削減する
* レスポンス時間を短縮する
* 帯域幅の使用を最適化する
* オフライン時の代替手段を提供する

<div id="deep-comparison">
  ### ディープ比較
</div>

高度なデータ比較機能により、以下が可能になります。

* ネストされたオブジェクト内の変更検出
* 効率的な状態管理
* 適切な更新トリガー
* UIアプリケーションでの再レンダリングの削減

<div id="typescript-support">
  ## TypeScript サポート
</div>

この SDK は TypeScript で記述されており、完全な型定義が用意されています。

```typescript
import { Session } from '@benzinga/session';
import { CalendarData, DividendEvent } from '@benzinga/calendar-data';

// TypeScriptはオートコンプリートと型チェックを提供します
const session = new Session({ apiKey: 'YOUR_API_KEY' });
const calendar = new CalendarData(session);

// 型安全なAPI呼び出し
const dividends: DividendEvent[] = await calendar.getDividends({
  dateFrom: '2024-01-01',
  dateTo: '2024-12-31',
  ticker: 'AAPL'
});
```

<div id="real-time-data-streams">
  ### リアルタイムデータストリーム
</div>

```typescript
import { Session } from '@benzinga/session';
import { QuoteStream } from '@benzinga/quotes';

const session = new Session({ apiKey: 'YOUR_API_KEY' });
const quoteStream = new QuoteStream(session);

// リアルタイムクオートを購読する
quoteStream.subscribe(['AAPL', 'MSFT', 'GOOGL'], (quote) => {
  console.log('Quote update:', quote);
});

// 完了したら購読を解除する
quoteStream.unsubscribe(['AAPL']);
```

<div id="error-handling">
  ## エラー処理
</div>

try-catch ブロックを使用して、エラーを適切に処理してください。

```typescript
import { Session } from '@benzinga/session';
import { NewsData } from '@benzinga/news-data';

const session = new Session({ apiKey: 'YOUR_API_KEY' });
const news = new NewsData(session);

try {
  const articles = await news.getNews({ pageSize: 10 });
  console.log(articles);
} catch (error) {
  if (error.code === 'UNAUTHORIZED') {
    console.error('Invalid API key');
  } else if (error.code === 'RATE_LIMIT') {
    console.error('Rate limit exceeded');
  } else {
    console.error('An error occurred:', error.message);
  }
}
```

<div id="pagination">
  ## ページネーション
</div>

ページ分割された結果を効率的に扱います:

```typescript
import { Session } from '@benzinga/session';
import { NewsData } from '@benzinga/news-data';

const session = new Session({ apiKey: 'YOUR_API_KEY' });
const news = new NewsData(session);

// 複数ページを取得
let page = 0;
let allArticles = [];

while (page < 5) {
  const articles = await news.getNews({
    page: page,
    pageSize: 100
  });

  if (articles.length === 0) break;

  allArticles = allArticles.concat(articles);
  page++;
}

console.log(`Fetched ${allArticles.length} articles`);
```

<div id="caching-strategy">
  ## キャッシュ戦略
</div>

パフォーマンス向上のため、組み込みキャッシュを活用してください。

```typescript
import { Session } from '@benzinga/session';
import { CalendarData } from '@benzinga/calendar-data';

const session = new Session({
  apiKey: 'YOUR_API_KEY',
  cache: {
    enabled: true,
    ttl: 300000  // 5分間キャッシュ
  }
});

const calendar = new CalendarData(session);

// 最初の呼び出しはAPIにアクセス
const earnings1 = await calendar.getEarnings({ ticker: 'AAPL' });

// 2回目の呼び出しはキャッシュされたデータを使用（TTL以内の場合）
const earnings2 = await calendar.getEarnings({ ticker: 'AAPL' });
```

<div id="best-practices">
  ## ベストプラクティス
</div>

<div id="1-reuse-session-objects">
  ### 1. Session オブジェクトを再利用する
</div>

セッションインスタンスを 1 つ作成し、アプリケーション全体で再利用します。

```typescript
// session.ts
export const globalSession = new Session({ apiKey: process.env.BENZINGA_API_KEY });

// 他のファイルから使用する場合
import { globalSession } from './session';
const news = new NewsData(globalSession);
```

<div id="2-environment-variables">
  ### 2. 環境変数
</div>

API キーは環境変数に安全に保存してください。

```typescript
// .envファイル
BENZINGA_API_KEY=your_api_key_here

// コード内
const session = new Session({
  apiKey: process.env.BENZINGA_API_KEY
});
```

<div id="3-type-safety">
  ### 3. 型安全性
</div>

型安全なAPI連携には、TypeScriptを活用してください：

```typescript
import type { NewsArticle, NewsParams } from '@benzinga/news-data';

const params: NewsParams = {
  pageSize: 10,
  displayOutput: 'full',
  ticker: 'AAPL'
};

const articles: NewsArticle[] = await news.getNews(params);
```

<div id="4-error-boundaries">
  ### 4. エラーバウンダリー
</div>

本番環境ではエラーバウンダリーを実装してください。

```typescript
const safeApiCall = async (fn: () => Promise<any>) => {
  try {
    return await fn();
  } catch (error) {
    console.error('API Error:', error);
    // エラー追跡サービスにログを記録
    return null;
  }
};

const articles = await safeApiCall(() => news.getNews({ pageSize: 10 }));
```
