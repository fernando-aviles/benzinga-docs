<div id="sdk-installation">
  ## SDK のインストール
</div>

<div id="getting-started">
  ### はじめに
</div>

JDK 11 以降が必要です。

以下のサンプルは、公開済みのSDKアーティファクトの使用方法を示しています。

Gradle:

```groovy
implementation 'org.benzinga:BZClient:0.2.7'
```

Maven:

```xml
<dependency>
    <groupId>org.benzinga</groupId>
    <artifactId>BZClient</artifactId>
    <version>0.2.7</version>
</dependency>
```

<div id="how-to-build">
  ### ビルド方法
</div>

git リポジトリをファイルシステムにクローンしたら、*nix 系システムでは `./gradlew build`、Windows では `gradlew.bat` を実行することで、ソースから SDK アーティファクトを `build` ディレクトリにビルドできます。

ソースからビルドし、SDK アーティファクトをローカルの Maven リポジトリ (ファイルシステム上) に公開する場合は、次のコマンドを使用します (git リポジトリをローカルにクローンした後) 。

*nix の場合:

```bash
./gradlew publishToMavenLocal -Pskip.signing
```

Windows では:

```bash
gradlew.bat publishToMavenLocal -Pskip.signing
```

<div id="sdk-example-usage">
  ## SDK 使用例
</div>

### 例

```java
package hello.world;

import java.lang.Exception;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.operations.GetAnalystReportsRawTextDataResponse;

public class Application {

    public static void main(String[] args) throws Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .page(700347L)
                .pagesize(558834L)
                .call();

        if (res.modelsAnalystReportRawTexts().isPresent()) {
            // レスポンスを処理する
        }
    }
}
```

<div id="retries">
  ## 再試行
</div>

この SDK の一部のエンドポイントは再試行をサポートしています。設定を行わずに SDK を使用した場合は、API が提供するデフォルトの再試行戦略が適用されます。ただし、デフォルトの再試行戦略は、操作ごとに、または SDK 全体に対して上書きできます。

単一の API 呼び出しに対するデフォルトの再試行戦略を変更するには、`retryConfig` ビルダーメソッドを使用して `RetryConfig` オブジェクトを指定します。

```java
package hello.world;

import java.lang.Exception;
import java.util.concurrent.TimeUnit;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.operations.GetAnalystReportsRawTextDataResponse;
import org.benzinga.BZClient.utils.BackoffStrategy;
import org.benzinga.BZClient.utils.RetryConfig;

public class Application {

    public static void main(String[] args) throws Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .retryConfig(RetryConfig.builder()
                    .backoff(BackoffStrategy.builder()
                        .initialInterval(1L, TimeUnit.MILLISECONDS)
                        .maxInterval(50L, TimeUnit.MILLISECONDS)
                        .maxElapsedTime(1000L, TimeUnit.MILLISECONDS)
                        .baseFactor(1.1)
                        .jitterFactor(0.15)
                        .retryConnectError(false)
                        .build())
                    .build())
                .page(700347L)
                .pagesize(558834L)
                .call();

        if (res.modelsAnalystReportRawTexts().isPresent()) {
            // レスポンスを処理する
        }
    }
}
```

再試行をサポートするすべての操作でデフォルトの再試行戦略を変更する場合は、SDK の初期化時に設定を指定できます。

```java
package hello.world;

import java.lang.Exception;
import java.util.concurrent.TimeUnit;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.operations.GetAnalystReportsRawTextDataResponse;
import org.benzinga.BZClient.utils.BackoffStrategy;
import org.benzinga.BZClient.utils.RetryConfig;

public class Application {

    public static void main(String[] args) throws Exception {

        Bzclient sdk = Bzclient.builder()
                .retryConfig(RetryConfig.builder()
                    .backoff(BackoffStrategy.builder()
                        .initialInterval(1L, TimeUnit.MILLISECONDS)
                        .maxInterval(50L, TimeUnit.MILLISECONDS)
                        .maxElapsedTime(1000L, TimeUnit.MILLISECONDS)
                        .baseFactor(1.1)
                        .jitterFactor(0.15)
                        .retryConnectError(false)
                        .build())
                    .build())
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .page(700347L)
                .pagesize(558834L)
                .call();

        if (res.modelsAnalystReportRawTexts().isPresent()) {
            // レスポンスを処理する
        }
    }
}
```

<div id="error-handling">
  ## エラー処理
</div>

このSDKのエラー処理は、基本的には想定どおりです。すべての操作は、レスポンスオブジェクトを返すか、例外をスローします。

デフォルトでは、APIエラーが発生すると `models/errors/APIException` 例外がスローされます。ある操作に対してカスタムのエラーレスポンスが指定されている場合、SDK はそれに対応する例外をスローすることもあります。各操作で発生する可能性がある例外の種類の詳細については、SDK ドキュメント内の該当する *エラー* テーブルを参照してください。たとえば、`get` メソッドは次の例外をスローします。

| エラータイプ                         | ステータスコード | コンテンツタイプ         |
| ------------------------------ | -------- | ---------------- |
| models/errors/ApiErrorResponse | 400, 500 | application/json |
| models/errors/APIException     | 4XX, 5XX | */*              |

### 例

```java
package hello.world;

import java.lang.Exception;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.errors.ApiErrorResponse;
import org.benzinga.BZClient.models.operations.GetAnalystInsightsV1Request;
import org.benzinga.BZClient.models.operations.GetAnalystInsightsV1Response;

public class Application {

    public static void main(String[] args) throws ApiErrorResponse, Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystInsightsV1Request req = GetAnalystInsightsV1Request.builder()
                .build();

        GetAnalystInsightsV1Response res = sdk.analystInsights().get()
                .request(req)
                .call();

        if (res.modelsAnalystInsightsJSON().isPresent()) {
            // レスポンスを処理する
        }
    }
}
```

<div id="select-server-by-index">
  ### インデックスでサーバーを選択
</div>

SDK クライアントインスタンスの初期化時に `.serverIndex(int serverIdx)` ビルダーメソッドを使用すると、デフォルトサーバーをグローバルに上書きできます。選択したサーバーは、それを使用する操作のデフォルトとして使用されます。以下の表は、利用可能なサーバーに対応するインデックスを示しています。

| # | サーバー                                |
| - | ----------------------------------- |
| 0 | `https://api.benzinga.com`          |
| 1 | `https://api.benzinga.com/api/v1`   |
| 2 | `https://api.benzinga.com/api/v2`   |
| 3 | `https://api.benzinga.com/api/v2.1` |
| 4 | `https://api.benzinga.com/api/v2.2` |

  #### 例
</div>

```java
package hello.world;
```

<div id="override-server-url-per-client">
  ### クライアントごとにサーバーURLを上書きする
</div>

デフォルトのサーバーは、SDKクライアントインスタンスの初期化時に `.serverURL(String serverUrl)` ビルダーメソッドを使用することで、グローバルに上書きすることもできます。例:

```java
package hello.world;

import java.lang.Exception;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.operations.GetAnalystReportsRawTextDataResponse;

public class Application {

    public static void main(String[] args) throws Exception {

        Bzclient sdk = Bzclient.builder()
                .serverURL("https://api.benzinga.com")
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .page(700347L)
                .pagesize(558834L)
                .call();
```
