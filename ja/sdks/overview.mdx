ローカル開発から本番環境まで、Benzinga SDK を使用すると、アプリで必要なときに Benzinga からシークレットを簡単に取得できます。

* 言語固有のクライアント SDK をアプリケーションにインストールして初期化する
* Benzinga で、プロジェクトと環境に対するスコープ付きアクセスをクライアントにプロビジョニングする
* 必要なときにシークレットを取得する

  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="API 呼び出しのたびにデータをリクエストするのは効率的ですか？">
    Benzinga API SDK はパフォーマンスが最適化されています。頻繁にアクセスされる data をキャッシュし、重複するリクエストを減らす仕組みにより、アプリケーションを効率的に動作させることができます。キャッシュの詳細は、SDK の実装や使用する言語によって異なる場合があります。
  </Accordion>

  <Accordion title="Benzinga API へのリクエストが失敗した場合はどうなりますか？">
    SDK にはエラーハンドリングと再試行の仕組みが組み込まれています。リクエストが失敗した場合、SDK は設定に応じて再試行を行うか、適切なエラーメッセージを返すため、アプリケーション側で障害に適切に対処できます。
  </Accordion>

  <Accordion title="アプリケーションの要件に合わせて SDK の設定をカスタマイズできますか？">
    もちろんです。Benzinga API SDK では、API キーの設定、タイムアウト設定、そのほかユースケースに応じたオプションなどのパラメータでカスタマイズできます。設定可能なパラメータの詳細については、ご利用の言語向けの SDK ドキュメントを参照してください。
  </Accordion>
</AccordionGroup>
