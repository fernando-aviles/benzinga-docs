ローカル開発から本番環境まで、Benzinga SDK は、アプリケーションで必要なときに Benzinga からシークレットを取得するための最も簡単な方法です。

* 言語固有のクライアント SDK をアプリケーションにインストールして初期化する
* クライアントに、Benzinga のプロジェクトおよび環境にスコープされたアクセス権を設定する
* 必要なときにシークレットを取得する

<div id="faq">
  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="API を呼び出すたびにデータをリクエストするのは効率的ですか？">
    Benzinga API SDK は、パフォーマンスを考慮して最適化されています。頻繁に参照されるデータをキャッシュし、不要な重複リクエストを減らす仕組みを備えているため、アプリケーションを効率的に動作させることができます。キャッシュの詳細は、SDK の実装や使用する言語によって異なる場合があります。
  </Accordion>

  <Accordion title="Benzinga API へのリクエストが失敗した場合はどうなりますか？">
    SDK にはエラー処理と再試行の仕組みが含まれています。リクエストが失敗した場合、SDK は設定に応じて再試行を行うか、適切なエラーメッセージを返すため、アプリケーション側で障害に適切に対処できます。
  </Accordion>

  <Accordion title="アプリケーションの要件に合わせて SDK の設定をカスタマイズできますか？">
    はい。Benzinga API SDK では、API キーの設定、タイムアウト設定、そのほかユースケースに応じたオプションなどのパラメータを通じてカスタマイズできます。設定可能なパラメータの詳細については、使用している言語向けの SDK ドキュメントを参照してください。
  </Accordion>
</AccordionGroup>
