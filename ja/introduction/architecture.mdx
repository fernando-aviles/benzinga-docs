Benzinga は、世界中のアプリケーションやサービスにリアルタイムの金融データを配信するための、堅牢かつ高い拡張性を備えた API 基盤を提供しています。速度と可用性を重視した Benzinga のアーキテクチャは、**Go (Golang)**、**Kafka**、**WebSockets**、**Elasticsearch**、**マイクロサービス** などの最新技術を活用し、ユーザーが必要なときにいつでも、信頼性が高く低レイテンシなデータを受け取れるよう設計されています。

<div id="high-performance-architecture-with-go-and-kafka">
  ## Go と Kafka による高性能アーキテクチャ
</div>

**Go (Golang)** の並行処理性能と高速性を活かして構築された Benzinga のインフラストラクチャは、大量の金融データトラフィックを効率的に処理します。データパイプラインとリアルタイムストリーミングは **Kafka** によって支えられており、低レイテンシでの高速なデータ取り込み、処理、配信を実現します。このアーキテクチャにより、グローバル展開においても拡張性と堅牢性が確保され、リアルタイム金融データ配信の要件に対応できます。

<div id="real-time-data-access-via-websockets-and-rest-apis">
  ## WebSocket と REST API によるリアルタイムデータへのアクセス
</div>

Benzinga の API は **WebSocket** と **REST** の両方のプロトコルをサポートしており、クライアントはリアルタイムデータにアクセスできるほか、オンデマンドでリクエストを送信できます。WebSocket 接続は、ライブの市場クオートや取引データなど、継続的なデータ配信を必要とするアプリケーションに最適です。一方、REST API は、過去データ、会社のファンダメンタルズ、ニュース、分析など、幅広いリクエストに対応します。

<div id="api-highlights">
  ### API の主な特長
</div>

* **リアルタイム市場データ**: リアルタイムの気配値、約定、その他の重要な市場データをストリーミング配信します。
* **過去データ**: バーデータ、気配値、経時的な市場パフォーマンスなどの過去データにアクセスできます。
* **ニュースとインサイト**: リアルタイムのニュースや相場を動かすインサイトを常に把握できます。
* **会社ファンダメンタルズ**: 売上高、利益、主要な比率などの重要な財務指標を取得できます。

<div id="data-storage-and-search-efficiency-with-elasticsearch">
  ## Elasticsearch によるデータ保存と検索の効率化
</div>

効率的なデータの取得と保存を実現するために、Benzinga では検索およびインデックス作成システムの中核コンポーネントとして **Elasticsearch** を採用しています。Elasticsearch により、構造化データと非構造化データの両方に高速にアクセスでき、履歴データやニュースアーカイブなどの大規模なデータセットに対しても、ユーザーはミリ秒単位の応答時間で迅速にクエリを実行できます。この構成は分析にも対応しており、クライアントはデータを分析して有益なインサイトを抽出できます。

<div id="microservices-based-infrastructure-for-scalability">
  ## スケーラビリティを支えるマイクロサービスベースのインフラ
</div>

Benzingaのアーキテクチャは、**マイクロサービス**アプローチを基盤としており、各サービスがデータの取り込み、処理、保存、配信など、それぞれ特定の機能を担います。このモジュール型のアプローチによりスケーラビリティが向上し、各コンポーネントを需要に応じて個別にスケールできます。さらに、マイクロサービスは軽量なプロトコルを介して通信するため、システム全体で低レイテンシと耐障害性を実現しています。

<div id="cdn-backed-delivery-for-images-and-static-content">
  ## 画像および静的コンテンツのCDN配信
</div>

画像、ロゴ、その他の静的アセットを迅速に配信するために、Benzingaでは**CDN** (コンテンツ配信ネットワーク) を採用しています。この仕組みにより、ユーザーに近い場所で静的リソースがキャッシュされるため、読み込み時間が短縮され、レイテンシも最小限に抑えられます。その結果、地理的な場所に関係なく、画像やファイルをほぼ瞬時に読み込めます。

<div id="key-benefits-of-benzingas-architecture">
  ## Benzingaのアーキテクチャの主な利点
</div>

* **スケーラビリティ**: 水平スケーリングを前提に設計されており、1分あたり数百万件のリクエストと多数の同時 WebSocket 接続をサポートします。
* **高可用性**: 冗長化されたシステムと地理的に分散したコンポーネントにより、高い稼働率とデータ可用性を確保します。
* **データ整合性**: Kafka と Elasticsearch を組み合わせることで、一貫したデータフローとデータ取得を実現し、信頼性の高いフェイルオーバー機構を提供します。
* **低レイテンシ**: Go の高い性能、Kafka のストリーミング、WebSocket のリアルタイム性を活用し、データを低レイテンシで配信できるよう最適化されています。

<div id="deployment-flexibility-and-infrastructure-integration">
  ## デプロイの柔軟性とインフラ統合
</div>

Benzinga のアーキテクチャは、クライアントごとの多様なニーズに対応できる柔軟なデプロイオプションをサポートしています。

* **Managed Cloud Service**: シームレスな運用を求めるクライアント向けに、Benzinga は自動スケーリングと保守を含むフルマネージドのクラウドサービスを提供しています。
* **Self-Hosted Option**: より高い制御性が必要な場合は、Benzinga のアーキテクチャをオンプレミス環境やプライベートクラウド環境向けに適応でき、完全なデータ所有権と各地域の規制への準拠を確保できます。

<div id="global-reach-with-edge-caching">
  ### エッジキャッシュによるグローバル展開
</div>

パフォーマンスをさらに向上させるため、Benzinga では **エッジキャッシュ** を活用し、リアルタイムストリームや静的コンテンツを含むデータを、最小限の遅延でユーザーに届けています。世界各地に Point of Presence (PoP) を戦略的に配置することで、Benzinga は各大陸のクライアントに対して高可用性と低レイテンシを実現しています。

この分散アーキテクチャにより、Benzinga は地理的な場所を問わず、信頼性の高いリアルタイムデータをクライアントに提供でき、ローカルユーザーとグローバルユーザーの双方のニーズに対応しています。
