<div id="benzinga-apis">
  ## Benzinga API
</div>

決算レポートからオプション取引動向まで、Benzinga は市場を動かす情報に注力しています。お客様の企業がいち早く取引を確定できるよう、速報ニュースに特化した複数の配信チャネルを提供しています。ニュースの受信方法は、プル型の REST API とプッシュ型の TCP の 2 つです。どちらも市場への比類のないアクセスを提供します。

Benzinga の基本理念の 1 つは、「競合ではなく顧客を徹底的に重視する」ことです。

良好な関係こそがビジネスの成功を支える原動力であり、それを常に最優先にしています。

Benzinga の API の利用を開始するには、licensing@benzinga.com まで直接メールでお問い合わせいただくか、877-440-9464 までお電話ください。1 営業日以内にご返信します。

<div id="authentication">
  ## 認証
</div>

リクエストのクエリ文字列に、API キー/トークンを token パラメータとして含めてください。

<div id="best-practices">
  ## ベストプラクティス
</div>

可能であれば、Benzinga 提供のクライアントを使用することをお勧めします。多くのエンドポイントは、当社の [Python ライブラリ](/ja/python-client) で利用できます。また、公開 [GitHub](https://github.com/benzinga) では、参照実装とクライアントライブラリを提供しています。

最後に、Calendar、Signals、News API の利用時には、特にリアルタイム取り込み (過去データの取り込みではなく) を行う場合、差分取得を強く推奨します。これは、Calendar API と Signals API では `parameters[updated]` クエリパラメータを、News API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシを最小限に抑えられます。

<div id="max-offset">
  ## 最大オフセット
</div>

News、Calendar、Signals の各 API のページネーションでは、10,000 件を超える結果セットを返すことはできません。つまり、ページサイズが 1,000 の場合、ページ 10 はリクエストできません。

この制限を回避するには、クエリの範囲を調整してください。たとえば、結果が 10,000 件未満になるよう、特定の日付範囲やティッカーのセットに絞り込めます。その後、同様に 10,000 件未満となる別の日付範囲やティッカーのセットを対象にクエリを調整し、それらのセットを結合する、という手順を、必要な結果セット全体を取得できるまで繰り返します。

<div id="return-format">
  ## 返却形式
</div>

Benzinga API のデフォルトの形式は各 API 仕様で定義されていますが、特に指定がない限り `JSON` です。

`accept` ヘッダーを `application/json` に設定すると `JSON` を要求でき、非推奨でない場合は `application/xml` を指定して `XML` を要求することもできます。

**`XML` 出力形式を使用する新規インテグレーションは推奨されません。**

<div id="example">
  ### 例
</div>

```JSON
curl -X GET \
'http://api.benzinga.com/api/v2/calendar/ratings?token=YOUR_TOKEN_HERE&ratings=Upgrades' \
-H 'accept: application/json'
```
