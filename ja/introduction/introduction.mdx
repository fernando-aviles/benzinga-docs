<div id="benzinga-apis">
  ## Benzinga APIs
</div>

決算レポートからオプション取引動向まで、Benzingaは市場を動かす情報に注力しています。お客様がいち早く取引機会を捉えられるよう、速報ニュースに特化した配信チャネル群を提供しています。ニュースの受信方法は、プル型のREST APIとプッシュ型のTCPの2通りです。どちらも市場へのこれまでにないアクセスを提供します。

Benzingaの基本理念の1つは、「競合ではなく顧客に徹底的に向き合う」ことです。

良好な関係こそがビジネス成功の原動力であり、私たちはこれを常に最優先にしています。

BenzingaのAPIの利用開始については、licensing@benzinga.com まで直接メールでご連絡いただくか、877-440-9464 までお電話ください。1営業日以内にご返信します。

<div id="authentication">
  ## 認証
</div>

API キーまたはトークンを、リクエストのクエリ文字列の token パラメータとして含めてください。

<div id="best-practices">
  ## ベストプラクティス
</div>

可能であれば、Benzinga 提供のクライアントを使用することを推奨します。多くのエンドポイントは、[Python library](/ja/python-client) で利用できます。リファレンス実装とクライアントライブラリは、公開中の [GitHub](https://github.com/benzinga) で提供しています。

最後に、Calendar、Signals、News API を使用する際は、特に履歴データの取り込みではなくリアルタイム取り込みに利用する場合、差分の活用を強く推奨します。これは、Calendar API と Signals API では `parameters[updated]` クエリパラメータを、News API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシを最小限に抑えられます。

<div id="max-offset">
  ## 最大オフセット
</div>

ニュース、カレンダー、Signals API のページネーションでは、10,000 件を超える結果セットは返却できません。つまり、ページサイズが 1,000 の場合、page 10 はリクエストできません。

この制限を回避するには、クエリサイズを調整してください。たとえば、件数が 10,000 件未満になる特定の date 範囲またはティッカーのセットに絞り込むことができます。その後、10,000 件未満の別の date 範囲またはティッカーのセットを含めるようにクエリを調整し、それらのセットを結合する、という手順を、必要な結果セット全体を取得するまで繰り返します。

<div id="return-format">
  ## 返却形式
</div>

Benzinga APIs のデフォルトの形式は各 API 仕様で定義されていますが、特に指定がない限り `JSON` です。

`accept` header に `application/json` を設定すると `JSON` をリクエストできます。また、非推奨でない場合は `application/xml` を指定して `XML` をリクエストすることもできます。

**`XML` 出力形式を使用する新規統合は推奨されません。**

<div id="example">
  ### 例
</div>

```JSON
curl -X GET \
'http://api.benzinga.com/api/v2/calendar/ratings?token=YOUR_TOKEN_HERE&ratings=Upgrades' \
-H 'accept: application/json'
```
