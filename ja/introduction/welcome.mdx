この [Benzinga Cloud](https://cloud.benzinga.com/) 開発者向けドキュメントポータルでは、ニュース、市場、企業データの API に関するドキュメントを参照できます。ご不明な点がありましたら、[お気軽にお問い合わせ](mailto:partners@benzinga.com)ください。喜んでお手伝いします。

<div id="about-this-documentation">
  ## このドキュメントについて
</div>

REST API のドキュメントは、OpenAPI 仕様に基づいて生成されています。生の YAML spec は、このドキュメントアプリケーションの公開リポジトリで確認できます。[こちら](https://github.com/Benzinga/doc-site-mintlify)をご覧ください。

OpenAPI に対応した各ページの「Playground」セクションには、統合された REST クライアントが直接埋め込まれています。API キーを取得すれば、ドキュメントポータルを離れることなく API 呼び出しをテストできます。また、他の GUI REST クライアント ([Postman](https://www.getpostman.com/)、[Insomnia](https://insomnia.rest/)、[Paw](https://paw.cloud/)) や、cUrl1 のような CLI ツールを使ってテストすることもできます。

最後に、OpenAPI 仕様は Swagger エコシステムの各種ツールと互換性があるため、[任意のプログラミング言語向けのクライアントを生成する](http://swagger.io/swagger-codegen/)こともできます。

<div id="best-practices">
  ## ベストプラクティス
</div>

可能であれば、Benzinga が提供するクライアントの使用をお勧めします。多くのエンドポイントは、当社の [Python library](https://github.com/Benzinga/benzinga-python-client) で利用できます。また、公開 [Github](https://github.com/benzinga) では、実装例やクライアントライブラリも提供しています。

最後に、Calendar、Signals、ニュース API を使用する際には、特にリアルタイム取り込み (履歴データの取り込みではなく) で利用する場合、差分の利用を強く推奨します。これは、Calendar API と Signals API では `parameters[updated]` クエリパラメータを、ニュース API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシを最小限に抑えられます。
