この [Benzinga Cloud](https://cloud.benzinga.com/) 開発者向けドキュメントポータルでは、ニュース、マーケット、会社データの API ドキュメントをご覧いただけます。ご不明な点がありましたら、[お気軽にお問い合わせください](mailto:partners@benzinga.com)。喜んでサポートいたします。

<div id="about-this-documentation">
  ## このドキュメントについて
</div>

BenzingaのREST APIに関するこのドキュメントは、OpenAPI仕様に基づいて構成されています。生の yaml 仕様は、このドキュメントアプリケーションの公開リポジトリを[こちら](https://github.com/Benzinga/doc-site-mintlify)から参照できます。

OpenAPIに対応した各ページの「Playground」セクションには、統合RESTクライアントが直接埋め込まれています。APIキーを取得すれば、ドキュメントポータルを離れることなく呼び出しをテストできます。なお、ほかのGUI RESTクライアント ([Postman](https://www.getpostman.com/)、[Insomnia](https://insomnia.rest/)、[Paw](https://paw.cloud/)) や、cUrl1のようなCLIツールを使って呼び出しをテストすることもできます。

最後に、OpenAPI仕様はSwaggerエコシステムのツールと互換性があるため、[任意のプログラミング言語向けのクライアントを生成する](http://swagger.io/swagger-codegen/)こともできます。

<Info>
  このドキュメントはAIに対応しています。Ctrl + K (PC) または Cmd + K (Mac) で質問してみてください。
</Info>

<div id="acessing-the-data">
  ## データへのアクセス
</div>

<Steps>
  <Step title="アカウントにログイン">

  </Step>
</Steps>

<div id="best-practices">
  ## ベストプラクティス
</div>

可能であれば、Benzinga が提供するクライアントの利用をお勧めします。多くのエンドポイントは [Python ライブラリ](https://github.com/Benzinga/benzinga-python-client) で利用できます。また、公開している [GitHub](https://github.com/benzinga) では、実装例やクライアントライブラリも提供しています。

最後に、Calendar、Signals、News API を利用する際は、差分更新を活用することを強く推奨します。特に、過去データの取り込みではなくリアルタイム取り込みに使用する場合に重要です。これは、Calendar API と Signals API では `parameters[updated]` クエリパラメータを、News API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシを最小限に抑えられます。
