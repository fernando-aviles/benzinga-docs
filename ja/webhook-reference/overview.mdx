---
title: "Webhook の概要"
description: "耐障害性が高くフィルタ可能な webhook を使って、Benzinga の市場データをお使いのシステムにプッシュ配信します。"
---

Benzinga の webhook は、リアルタイムの市場イベントを HTTP エンドポイントに直接送信することで、ポーリングを不要にします。配信は JSON の `POST` リクエストとして行われ、重複排除のための一意の `X-BZ-Delivery` ヘッダーを含み、信頼性を最大限に高めるために段階的な再試行戦略を採用しています。

<div id="available-webhook-services">
  ## 利用可能なWebhookサービス
</div>

* **Data Webhook Engine**: 設定可能なフィルターとオプションのペイロード変換に対応した、リアルタイムのカレンダー／シグナルデータ (決算、配当、レーティング、オプションアクティビティ、WIIMs など) を提供します。詳しくは [Data Webhook Engine](./webhook-engine) を参照してください。
* 追加のWebhook設定は、アカウント単位で利用できるように設定できます。特定のデータセットを有効にする場合や、新しいWebhook URLを登録する場合は、Benzingaの担当者にお問い合わせください。

<div id="integration-flow">
  ## 連携フロー
</div>

<Steps>
  <Step title="エンドポイントとフィルターを共有">
    外部からアクセス可能な HTTPS エンドポイントと、必要なデータタイプまたは地域のフィルターを指定してください。Benzinga はその設定に基づいて Webhook をプロビジョニングします。
  </Step>

  <Step title="連携をテスト">
    [Test Webhook Delivery](/ja/api-reference/webhook_api/test-webhook-delivery) エンドポイントを使用して、Webhook エンドポイントが適切に構成されており、配信を受信できることを確認します。
  </Step>

  <Step title="冪等に配信を処理">
    JSON ペイロードを解析し、重複を回避するために `id` と `X-BZ-Delivery` の値を使用し、受信確認として 2xx ステータスですみやかに応答してください。
  </Step>

  <Step title="監視して調整">
    失敗、タイムアウト、再試行をログで追跡します。下流システムに新たな要件がある場合は、Benzinga の担当者と連携してフィルターやペイロード変換を調整してください。
  </Step>
</Steps>

<div id="best-practices">
  ## ベストプラクティス
</div>

* Webhook エンドポイントでは、HTTPS と認証を使用してください。
* レスポンスが 30 秒以内に返るよう、処理は非同期で行ってください。
* 再試行時の冪等性を確保するため、処理済みの配信 ID を保存してください。
* 本番配信に切り替える前に、非本番環境のエンドポイントでテストしてください。
