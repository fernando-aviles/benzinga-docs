---
title: "ウェブフック概要"
description: "耐障害性が高く、フィルタリング可能なウェブフックを使用して、Benzingaの市場データをお使いのシステムにプッシュ配信します。"
---

Benzingaのウェブフックは、リアルタイムの市場イベントをHTTPエンドポイントに直接POSTすることで、ポーリングを不要にします。配信は JSON `POST` リクエストとして行われ、重複排除のための一意な `X-BZ-Delivery` ヘッダーが含まれます。また、信頼性を最大限に高めるため、段階的な再試行戦略を採用しています。

<div id="available-webhook-services">
  ## 利用可能なウェブフックサービス
</div>

* **Data Webhook Engine**: 設定可能なフィルターとオプションのペイロード変換に対応した、リアルタイムのカレンダーおよびシグナルデータ (決算、配当、レーティング、オプションアクティビティ、WIIMs など) 。詳細は [Data Webhook Engine](./webhook-engine) を参照してください。
* 追加のウェブフック設定は、お客様のアカウント単位で構成できます。特定のデータセットを有効にしたり、新しいウェブフック URL を登録したりするには、Benzinga の担当者にお問い合わせください。

<div id="integration-flow">
  ## 統合フロー
</div>

<Steps>
  <Step title="エンドポイントとフィルターを共有する">
    外部から到達可能な HTTPS エンドポイントと、必要なデータ種別または地域フィルターを指定してください。Benzinga がその設定でウェブフックを構成します。
  </Step>

  <Step title="統合をテストする">
    [Test Webhook Delivery](/ja/api-reference/webhook_api/test-webhook-delivery) エンドポイントを使用して、ウェブフックエンドポイントが正しく設定され、配信を受信できることを確認してください。
  </Step>

  <Step title="冪等に配信を処理する">
    JSON ペイロードを解析し、`id` と `X-BZ-Delivery` の値を使って重複を回避し、受信確認として 2xx ステータスですみやかに応答してください。
  </Step>

  <Step title="監視して改善を続ける">
    失敗、タイムアウト、再試行をログで追跡してください。ダウンストリームシステムに新しい要件がある場合は、Benzinga の担当者と連携してフィルターやペイロード変換を調整してください。
  </Step>
</Steps>

<div id="best-practices">
  ## ベストプラクティス
</div>

* ウェブフックエンドポイントでは、HTTPS と認証を使用してください。
* レスポンスを 30 秒以内に返せるよう、処理は非同期で行ってください。
* 再試行時にも冪等性を保てるよう、処理済みの配信 ID を保存してください。
* 本番配信に切り替える前に、非本番環境のエンドポイントでテストしてください。
