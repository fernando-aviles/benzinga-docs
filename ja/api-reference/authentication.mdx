title: "認証"
description: "Benzinga API へのリクエストを認証する方法"
---

Benzinga API では、リクエストの認証に **APIキー** を使用します。APIキーは一意の識別子であり、サブスクリプションに応じて特定のデータや機能へのアクセスが許可されます。

> [!IMPORTANT]
> **APIキーは厳重に管理してください。** GitHub、クライアントサイドのコード、保護されていない通信など、一般公開された場所で共有しないでください。キーが漏えいした可能性がある場合は、直ちにサポートまでご連絡ください。

APIキーは [Benzinga Developer Console](https://www.api.benzinga.com/token) で確認および管理できます。

<div id="authentication-methods">
  ## 認証方法
</div>

Benzinga API では、認証方法として 2 つの方式をサポートしています。**HTTP Header** 方式は、より安全で、URL ログにキーが表示されるのを防げるため、本番環境のアプリケーションではこの方式の使用を強く推奨します。

<div id="1-http-header-recommended">
  ### 1. HTTP Header (推奨)
</div>

ヘッダーで認証するには、`Authorization` ヘッダーに `token <YOUR_API_KEY>` を指定します。

```http
Authorization: token <YOUR_API_KEY>
```

<div id="2-query-parameter">
  ### 2. クエリパラメータ
</div>

手早くテストしたい場合や、ヘッダーを変更できない場合は、`token` という名前のクエリパラメータでキーを渡せます。

```http
https://api.benzinga.com/api/v2/news?token=<YOUR_API_KEY>
```

<div id="code-examples">
  ## コード例
</div>

以下は、一般的な言語で Benzinga API に接続するための本番環境向けの例です。

<CodeGroup>
  ```bash cURL
  # 推奨: ヘッダー認証
  curl -L 'https://api.benzinga.com/api/v2/news?pageSize=1' \
  -H 'Authorization: token YOUR_API_KEY' \
  -H 'Accept: application/json'

  # 代替: クエリパラメータ
  curl -L 'https://api.benzinga.com/api/v2/news?pageSize=1&token=YOUR_API_KEY'
  ```

  ```python Python
  import requests

  url = "https://api.benzinga.com/api/v2/news"

  # 推奨: ヘッダー認証
  headers = {
      "Authorization": "token YOUR_API_KEY",
      "Accept": "application/json"
  }

  params = {
      "pageSize": 1
  }

  try:
      response = requests.get(url, headers=headers, params=params)
      response.raise_for_status() # 4xx/5xx レスポンスに対して詳細なエラーを発生させる
      data = response.json()
      print(data)
  except requests.exceptions.RequestException as e:
      print(f"Error fetching data: {e}")
  ```

  ```javascript Node.js
  const axios = require('axios');

  const url = 'https://api.benzinga.com/api/v2/news';
  const apiKey = 'YOUR_API_KEY';

  async function getNews() {
    try {
      const response = await axios.get(url, {
        headers: {
          'Authorization': `token ${apiKey}`,
          'Accept': 'application/json'
        },
        params: {
          pageSize: 1
        }
      });
      
      console.log(response.data);
    } catch (error) {
      if (error.response) {
        // リクエストは送信され、サーバーからステータスコードが返されました
        // そのコードは 2xx の範囲外です
        console.error('Error Status:', error.response.status);
        console.error('Error Data:', error.response.data);
      } else {
        console.error('Error Message:', error.message);
      }
    }
  }

  getNews();
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"io/ioutil"
  	"net/http"
  )

  func main() {
  	url := "https://api.benzinga.com/api/v2/news?pageSize=1"
  	apiKey := "YOUR_API_KEY"

  	req, err := http.NewRequest("GET", url, nil)
  	if err != nil {
  		panic(err)
  	}

  	// 推奨: ヘッダー認証
  	req.Header.Add("Authorization", fmt.Sprintf("token %s", apiKey))
  	req.Header.Add("Accept", "application/json")

  	client := &http.Client{}
  	resp, err := client.Do(req)
  	if err != nil {
  		panic(err)
  	}
  	defer resp.Body.Close()

  	if resp.StatusCode >= 400 {
  		fmt.Printf("Error: Status Code %d\n", resp.StatusCode)
  		return
  	}

  	body, err := ioutil.ReadAll(resp.Body)
  	if err != nil {
  		panic(err)
  	}

  	fmt.Println(string(body))
  }
  ```
</CodeGroup>

<div id="troubleshooting">
  ## トラブルシューティング
</div>

よくある認証エラーと、その解決方法を示します。

| ステータスコード | メッセージ          | 考えられる原因   | 対処方法                                                                                     |
| :------- | :------------- | :-------- | :--------------------------------------------------------------------------------------- |
| **401**  | `Unauthorized` | 無効なAPIキー  | APIキーが正しく、再生成されていないことを確認してください。余分な空白が含まれていないことも確認してください。                                 |
| **401**  | `Unauthorized` | APIキーの未指定 | `Authorization` ヘッダーが `token <KEY>` の形式で正しく指定されているか、または `token` パラメータが含まれていることを確認してください。 |
| **403**  | `Forbidden`    | 権限不足      | APIキーは有効ですが、ご利用のプランには要求されたエンドポイントへのアクセス権が含まれていません。プランのアップグレードについては、営業またはサポートにお問い合わせください。 |
