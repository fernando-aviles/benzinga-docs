<div id="introduction">
  ## はじめに
</div>

Benzinga Logos API では、企業、ファンド、暗号資産の高品質なロゴを利用できます。シンボル、CIK 番号、ISIN コード、CUSIP、FIGI など、複数の識別子タイプを使ってロゴを検索できます。

<div id="available-endpoints">
  ## 利用可能なエンドポイント
</div>

<div id="logossearch">
  ### `/logos/search`
</div>

さまざまな識別子を使用してロゴを検索します。ロゴのURL、証券情報、メタデータが返されます。

<div id="logossync">
  ### `/logos/sync`
</div>

ページネーションをサポートした、複数のロゴを取得するための一括同期エンドポイント。

<div id="search-filters">
  ## 検索フィルター
</div>

このAPIでは、主に次の2つのパラメータで柔軟な検索を行えます。

* `search_keys` - 検索対象の識別子の値
* `search_keys_type` - 識別子の種別 (省略可能。デフォルトは `symbol`)

<div id="1-filter-by-symbol">
  ### 1. シンボルで絞り込む
</div>

株式のティッカーシンボルを使用してロゴを検索します。

**パラメータ:**

* `search_keys`: シンボル (例: `AAPL`, `TSLA`, `MSFT`) 
* `search_keys_type`: `symbol` (省略可能、デフォルト) 

**例:**

```
GET /api/v2/logos/search?search_keys=AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="2-filter-by-cik">
  ### 2. CIKでフィルタリング
</div>

SECのCentral Index Key (CIK) を使用して検索します。ゼロ埋めされた値とされていない値の両方に対応しています。

**パラメータ:**

* `search_keys`: CIK番号 (例: `320193` または `0000320193`) 
* `search_keys_type`: `cik`

**例:**

```
GET /api/v2/logos/search?search_keys=320193&search_keys_type=cik&fields=logo_light,logo_dark
```

**ゼロ埋めされた CIK の例:**

```
GET /api/v2/logos/search?search_keys=0000320193&search_keys_type=cik&fields=logo_light,logo_dark
```

<div id="3-filter-by-isin">
  ### 3. ISINで絞り込む
</div>

国際証券識別番号 (ISIN) を使用して検索します。

**パラメータ:**

* `search_keys`: ISINコード (例: `US0378331005`) 
* `search_keys_type`: `isin`

**例:**

```
GET /api/v2/logos/search?search_keys=US0378331005&search_keys_type=isin&fields=logo_light,logo_dark
```

<div id="4-filter-by-figi-share-class">
  ### 4. FIGIの株式クラスで絞り込む
</div>

BloombergのFinancial Instrument Global Identifier (FIGI) を使用して株式クラスを検索します。

**パラメータ:**

* `search_keys`: FIGI Share Class ID (例: `BBG001SQKGD7`) 
* `search_keys_type`: `figi_share_class`

**例:**

```
GET /api/v2/logos/search?search_keys=BBG001SQKGD7&search_keys_type=figi_share_class&fields=logo_light,logo_dark
```

<div id="5-filter-by-cusip">
  ### 5. CUSIPで絞り込む
</div>

CUSIP番号を使用して検索します。

**パラメータ:**

* `search_keys`: CUSIP番号 (例: `88160R101`) 
* `search_keys_type`: `cusip`

**例:**

```
GET /api/v2/logos/search?search_keys=88160R101&search_keys_type=cusip&fields=logo_light,logo_dark
```

<div id="6-filter-by-symbol-with-currency-country">
  ### 6. 通貨・国を指定してシンボルで絞り込む
</div>

区切り文字としてコロン (`:`) またはピリオド (`.`) を使用し、シンボルに国コードまたは通貨コードを付けて指定します。

**パラメータ:**

* `search_keys`: 国コードまたは通貨コード付きのシンボル (例: `AAPL:US`, `AAPL:USD`, `AAPL.US`) 
* `search_keys_type`: `symbol`

**例:**

```
GET /api/v2/logos/search?search_keys=AAPL:US&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL:USD&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL.US&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="7-filter-by-exchange-and-symbol">
  ### 7. 取引所とシンボルでフィルタリング
</div>

取引所コードとシンボルを組み合わせて検索します。Bloomberg形式 (旧形式) とLSEG形式の両方に対応しています。

**Bloomberg形式の対応関係 (旧形式) :**

* `search_keys`: 取引所とシンボル (例: `NASDAQ:AAPL`) 
* `search_keys_type`: `symbol`

**LSEG 3文字コード (現行) :**

* `search_keys`: 取引所コードとシンボル (例: `NSQ:AAPL`) 
* `search_keys_type`: `symbol`

**例:**

```
GET /api/v2/logos/search?search_keys=NASDAQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=NSQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="8-filter-by-iso-mic-and-symbol">
  ### 8. ISO MIC とシンボルで絞り込む
</div>

ISO Market Identifier Code とシンボルを組み合わせて検索します。

**パラメータ:**

* `search_keys`: MIC コードとシンボル (例: `XYNS:AAPL`) 
* `search_keys_type`: `symbol`

**例:**

```
GET /api/v2/logos/search?search_keys=XYNS:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="9-crypto-logo-filters">
  ### 9. 暗号資産ロゴのフィルター
</div>

さまざまな形式で暗号資産のロゴを検索できます。`search_keys_type` が指定されていない場合でも、API は暗号資産の形式を自動的に認識します。

**対応している形式:**

* `CRYPTO:BTC`
* `CRYPTO/BTC`
* `$BTC`
* `BTC/USD`

**例:**

```
GET /api/v2/logos/search?search_keys=CRYPTO:BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=CRYPTO/BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=$BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=BTC/USD&fields=logo_light,logo_dark
```

<div id="available-fields">
  ## 利用可能なフィールド
</div>

`fields` パラメータを使用して、特定のロゴタイプを指定できます。利用可能なフィールドは次のとおりです。

* `logo_light` - ライトテーマのロゴ (PNG)
* `logo_dark` - ダークテーマのロゴ (PNG)
* `logo_vector_light` - ライトテーマのベクターロゴ (SVG)
* `logo_vector_dark` - ダークテーマのベクターロゴ (SVG)
* `mark_light` - ライトテーマのマーク/アイコン (PNG)
* `mark_dark` - ダークテーマのマーク/アイコン (PNG)
* `mark_vector_light` - ライトテーマのベクターマーク (SVG)
* `mark_vector_dark` - ダークテーマのベクターマーク (SVG)
* `mark_composite_light` - ライトテーマのコンポジットマーク (PNG)
* `mark_composite_dark` - ダークテーマのコンポジットマーク (PNG)
* `mark_vector_composite_light` - ライトテーマのベクターコンポジットマーク (SVG)
* `mark_vector_composite_dark` - ダークテーマのベクターコンポジットマーク (SVG)

**複数のフィールドを指定する例:**

```
GET /api/v2/logos/search?search_keys=TSLA&fields=logo_light,logo_dark,mark_vector_composite_light
```

<div id="additional-parameters">
  ## 追加のパラメータ
</div>

<div id="image-customization">
  ### 画像のカスタマイズ
</div>

* `scale` - 画像のサイズを指定します (例: `100x100`、`300x300`) 
* `max_width` - 返却される画像の最大幅
* `composite_radius` - 合成画像の角丸半径 (整数、範囲 `0-50`) 
* `composite_auto` - 合成画像を自動生成します (ブール値)

<div id="pagination-bulk-sync-only">
  ### ページネーション (bulk-sync のみ)
</div>

* `page` - ページ番号 (整数) 
* `pagesize` - ページあたりの結果数 (整数) 
* `updated_since` - 更新タイムスタンプで絞り込み (ISO 8601 形式)

<div id="security-details">
  ### 証券の詳細
</div>

* `securities` - レスポンスに詳細な証券情報を含めるかどうか (真偽値)

<div id="authentication">
  ## 認証
</div>

すべての Logos API エンドポイントで、API キーによる認証が必要です。token をクエリパラメータとして指定してください。

```
?token=YOUR_API_KEY
```

<div id="best-practices">
  ## ベストプラクティス
</div>

1. **search&#95;keys&#95;type を指定する** - API のデフォルトは `symbol` ですが、種別を明示的に指定することで、可読性とパフォーマンスが向上します
2. **必要なフィールドのみをリクエストする** - 必要なロゴ種別だけをリクエストして、レスポンスサイズを最小限に抑え、パフォーマンスを向上させます
3. **大規模なデータセットには bulk-sync を使用する** - 複数のロゴを取得する場合は、ページネーション付きの `/logos/bulk-sync` エンドポイントを使用します
4. **レスポンスをキャッシュする** - ロゴ URL には有効期限のタイムスタンプが含まれているため、API 呼び出しを減らすために適切にキャッシュしてください
5. **エラーを適切に処理する** - レスポンスの `ok` フィールドを必ず確認し、エラー時も適切に処理してください
