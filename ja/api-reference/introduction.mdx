Benzinga API ドキュメントへようこそ。このガイドでは、HTTP リクエストを通じて Benzinga API を使用し、Benzinga のリソースを作成、取得、更新、削除する方法を説明します。

<div id="base-url">
  ## ベースURL
</div>

ドキュメント内で参照されるすべてのURLは、以下のベースURLを使用します。

```
https://api.benzinga.com
```

<div id="authentication">
  ## 認証
</div>

Benzinga API ではベアラー認証を使用します。リクエストを送信する際は、`token`
クエリパラメータに `token: <token>` の形式で API トークンを指定する必要があります。詳細については、[認証](./authentication) ページを参照してください。

<div id="errors">
  ## エラー
</div>

エラーが発生した場合、Benzinga API は一般的な HTTP レスポンスコードと、エラーの詳細を含む JSON オブジェクトを返します。詳細については、[エラー](./errors) ページを参照してください。

<div id="rate-limiting">
  ## レート制限
</div>

不正利用の防止とサービスの安定性確保のため、Benzinga API にはレート制限が設けられています。これらの制限を超えると、
リクエストはスロットリングされ、`429 Too Many Requests` レスポンスが返されます。
