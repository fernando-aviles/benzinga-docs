Benzinga API ドキュメントへようこそ。このガイドでは、HTTP リクエストを通じて Benzinga のリソースを作成、取得、更新、削除するための、Benzinga API との基本的なやり取りについて説明します。

<div id="base-url">
  ## ベースURL
</div>

ドキュメント内で参照されるすべてのURLのベースは、次のとおりです。

```
https://api.benzinga.com
```

<div id="authentication">
  ## 認証
</div>

Benzinga API ではベアラー認証を使用します。リクエストを送信する際は、`token`
クエリパラメータに `token: <token>` 形式で API token を含める必要があります。詳しくは、[認証](./authentication) ページを参照してください。

<div id="errors">
  ## エラー
</div>

エラーが発生した場合、Benzinga API は標準的な HTTP レスポンスコードと、エラーの詳細情報を含む JSON オブジェクトを返します。詳しくは、[エラー](./errors) ページを参照してください。

<div id="rate-limiting">
  ## レート制限
</div>

不正利用の防止とサービスの安定性確保のため、Benzinga API にはレート制限が設けられています。これらの制限を超えると、
リクエストは制限され、`429 Too Many Requests` レスポンスが返されます。
