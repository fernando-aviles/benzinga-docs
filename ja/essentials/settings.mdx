すべての Mintlify サイトでは、基本的な設定を記述した `docs.json` ファイルが必要です。詳しくは、以下の [プロパティ](#properties) を参照してください。

<div id="properties">
  ## プロパティ
</div>

<ResponseField name="name" type="string" required>
  プロジェクトの名前です。グローバルタイトルに使用されます。

  例: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  各グループに含まれるすべてのページをまとめた、グループの配列です

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      グループ名です。

      例: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      ページとして使用する Markdown ファイルへの相対パスです。

      例: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  ロゴ画像へのパス、または &quot;light&quot; モードと &quot;dark&quot; モードのロゴ画像パスを含むオブジェクトです

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      ライトモード用ロゴへのパスです
    </ResponseField>

    <ResponseField name="dark" type="string">
      ダークモード用ロゴへのパスです
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      ロゴをクリックしたときの遷移先です
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  ファビコン画像へのパスです
</ResponseField>

<ResponseField name="colors" type="Colors">
  グローバルテーマ用の16進数カラーコードです

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      メインカラーです。主に、ライトモードで強調コンテンツ、セクション
      見出し、アクセントに使用されます
    </ResponseField>

    <ResponseField name="light" type="string">
      ダークモード用のメインカラーです。主に、強調表示された
      コンテンツ、セクション見出し、アクセントにダークモードで使用されます
    </ResponseField>

    <ResponseField name="dark" type="string">
      重要なボタンに使用するメインカラーです
    </ResponseField>

    <ResponseField name="background" type="object">
      ライトモードとダークモード両方の背景色です

      <Expandable title="オブジェクト">
        <ResponseField name="light" type="string" required>
          ライトモードの背景の16進数カラーコードです
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          ダークモードの背景の16進数カラーコードです
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  トップバーに含めるリンクの `name` と `url` の配列です

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      ボタンの名前です。

      例: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      ボタンをクリックしたときの遷移先 url です。例: `https://mintlify.com/contact`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      Link はボタンを表示します。GitHub は、指定した url のリポジトリ情報と GitHub スター数を表示します。
    </ResponseField>

    <ResponseField name="url" type="string">
      `link` の場合: ボタンのリンク先です。

      `github` の場合: GitHub 情報を読み込むリポジトリへのリンクです。
    </ResponseField>

    <ResponseField name="name" type="string">
      ボタン内に表示するテキストです。`type` が `link` の場合にのみ必須です。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  バージョン名の配列です。ナビゲーションバーのドロップダウンで
  異なるバージョンのドキュメントを表示したい場合にのみ使用します。
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  アンカーの配列です。`icon`、`color`、`url` が含まれます。

  <Expandable title="アンカー">
    <ResponseField name="icon" type="string">
      アンカーを表すために使用する [Font Awesome](https://fontawesome.com/search?s=brands%2Cduotone) アイコンです。

      例: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      アンカーラベルの名前です。

      例: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      どのページをこのアンカーに含めるかを判定する URL の先頭部分です。通常は、ページを配置するフォルダ名になります。
    </ResponseField>

    <ResponseField name="color" type="string">
      アンカーアイコンの背景の 16 進カラーです。`from` と `to` をプロパティに持つオブジェクトを渡すと、各プロパティに 16 進カラーを指定してグラデーションにすることもできます。
    </ResponseField>

    <ResponseField name="version" type="string">
      正しいドキュメントバージョンが選択されるまでアンカーを非表示にしたい場合に使用します。
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      その中のドキュメントへの直接リンクが開かれるまでアンカーを非表示にしたい場合は、`true` を指定します。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot;, または &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  最上位アンカーのデフォルト設定を上書きします。

  <Expandable title="オブジェクト">
    <ResponseField name="name" default="Documentation" type="string">
      最上位アンカーの名前です
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome アイコンです。
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      次のいずれか: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot;, または &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  ナビゲーションタブの配列です。

  <Expandable title="タブ">
    <ResponseField name="name" type="string">
      タブラベルの名前です。
    </ResponseField>

    <ResponseField name="url" type="string">
      どのページをこのタブに含めるかを判定する URL の先頭部分です。通常は、
      ページを配置するフォルダ名になります。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 設定に関する構成です。API ページの詳細は [API Components](/ja/api-playground/demo) を参照してください。

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      すべての API エンドポイントのベース URL です。`baseUrl` が配列の場合、ユーザーが切り替えられる複数のベース URL
      オプションを有効にできます。
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          すべての API エンドポイントで使用する認証方式です。
        </ResponseField>

        <ResponseField name="name" type="string">
          API playground で使用する認証パラメータ名です。

          method が `basic` の場合、形式は `[usernameName]:[passwordName]` である必要があります。
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          認証入力フィールドの接頭辞として使用するためのデフォルト値です。

          例: `inputPrefix` に `AuthKey` を指定すると、認証フィールドのデフォルト入力値として `AuthKey` が使用されます。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API playground の構成です。

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          playground を表示するか、非表示にするか、または追加のユーザー操作なしでエンドポイントのみを表示する `simple` にするかを指定します。

          詳細は [playground guides](/ja/api-playground/demo) を参照してください。
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      このフラグを有効にすると、OpenAPI ページのキー順が OpenAPI ファイルで定義されたキー順と一致します。

      <Warning>この動作はまもなくデフォルトで有効になり、その時点でこのフィールドは非推奨になります。</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPI ファイルを指す URL または相対パスを指定する文字列、もしくはその文字列配列です。

  例:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  ソーシャルメディアアカウントのオブジェクトです。key:property の組み合わせは、ソーシャルメディアプラットフォームとアカウント URL を表します。

  例:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      次のいずれかの値です: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      例: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      ソーシャルプラットフォームの URL です。

      例: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  フィードバックボタンを有効にするための構成です。

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      ユーザーがプルリクエストを通じて編集を提案できるボタンを有効にします。
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      ユーザーがドキュメントに関する issue を報告できるボタンを有効にします。
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  ダークモードの切り替えをカスタマイズします。

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      新規ユーザーに常にライトモードまたはダークモードを表示したい場合に設定します。設定しない場合、デフォルトではユーザーのオペレーティングシステムと同じモードが適用されます。
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      ダーク/ライトモードの切り替えを非表示にするには、true に設定します。`isHidden` と `default` を組み合わせると、ドキュメントをライトモードまたはダークモードのどちらかのみに固定できます。例:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Only Light Mode
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  すべてのページの背景として表示される画像です。
  [Infisical](https://infisical.com/docs) および [FRPC](https://frpc.io) の例を参照してください。
</ResponseField>
