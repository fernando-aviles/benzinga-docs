ナビゲーションメニューは、どのWebサイトにもあるリンクの一覧です。

新しいページを追加するたびに、`docs.json` を更新する必要があります。ページは自動では表示されません。

<div id="navigation-syntax">
  ## ナビゲーション構文
</div>

ナビゲーション構文は再帰的になっているため、ネストされたナビゲーショングループを作成できます。ページ名に `.mdx` を含める必要はありません。タブの中にグループを配置し、グループの中にページを配置します。

<CodeGroup>
  ```json 通常のナビゲーション
  "navigation": {
    "tabs": [
      {
        "tab": "Documentation",
        "groups": [
          {
            "group": "Getting Started",
            "pages": ["quickstart"]
          }
        ]
      }
    ]
  }
  ```

  ```json ネストされたナビゲーション
  "navigation": {
    "tabs": [
      {
        "tab": "Documentation",
        "groups": [
          {
            "group": "Getting Started",
            "pages": [
              "quickstart",
              {
                "group": "Nested Reference Pages",
                "pages": ["nested-reference-page"]
              }
            ]
          }
        ]
      }
    ]
  }
  ```
</CodeGroup>

<div id="folders">
  ## フォルダ
</div>

MDX ファイルをフォルダに入れ、`docs.json` のパスを更新するだけです。

たとえば、`https://yoursite.com/your-folder/your-page` にページを作成する場合は、`your-folder` というフォルダを作成し、その中に `your-page.mdx` という MDX ファイルを配置します。

<Warning>
  別のフォルダ内にネストしない限り、フォルダ名に `api` は使用できません。Mintlify は Next.js を使用しており、最上位レベルの `api` フォルダは内部サーバー呼び出し用に予約されています。`api-reference` のようなフォルダ名であれば使用できます。
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Documentation",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

<div id="hidden-pages">
  ## 非表示ページ
</div>

`docs.json` に含まれていない MDX ファイルはサイドバーには表示されませんが、検索バーから、または直接リンクすることでアクセスできます。
