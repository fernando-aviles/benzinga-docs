この [Benzinga Cloud](https://cloud.benzinga.com/) 開発者向けドキュメントポータルでは、ニュース、市場、企業データのAPIに関するドキュメントをご覧いただけます。ご質問がありましたら、[こちらまでお問い合わせください](mailto:partners@benzinga.com)。喜んでお手伝いします。

<div id="about-this-documentation-and-the-new-features">
  ## このドキュメントと新機能について
</div>

REST API については、このドキュメントは OpenAPI 仕様に基づいて提供されています。生の YAML 仕様は、このドキュメントアプリケーションの公開リポジトリを [こちら](https://github.com/Benzinga/doc-site-mintlify) で確認できます。

OpenAPI ベースの各ページの「Playground」セクションには、統合 REST クライアントが直接埋め込まれています。API キーを取得すると、ドキュメントポータルを離れることなく呼び出しをテストできます。別の方法として、他の GUI REST クライアント ([Postman](https://www.getpostman.com/)、[Insomnia](https://insomnia.rest/)、[Paw](https://paw.cloud/)) や、cUrl1 のような CLI ツールでも呼び出しをテストできます。

最後に、OpenAPI 仕様は Swagger エコシステムのツールと互換性があるため、[任意のプログラミング言語向けのクライアントを生成する](http://swagger.io/swagger-codegen/)こともできます。

<div id="best-practices">
  ## ベストプラクティス
</div>

可能であれば、Benzinga が提供するクライアントの利用をお勧めします。多くのエンドポイントは、当社の [Python ライブラリ](https://github.com/Benzinga/benzinga-python-client) で利用できます。また、公開している [GitHub](https://github.com/benzinga) では、実装例やクライアントライブラリも提供しています。

最後に、Calendar、Signals、News API の利用時には、特にリアルタイム取り込み (履歴データの取り込みではなく) で使用する場合、差分取得の利用を強く推奨します。これは、Calendar API と Signals API では `parameters[updated]` クエリパラメータを、News API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシを最小限に抑えられます。
