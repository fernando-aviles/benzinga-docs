この [Benzinga Cloud](https://cloud.benzinga.com/) 開発者向けドキュメントポータルでは、ニュース、市場データ、company データのAPIに関するドキュメントをご覧いただけます。ご不明な点がありましたら、[こちらまでお問い合わせください](mailto:partners@benzinga.com)。お手伝いします。

<div id="about-this-documentation">
  ## このドキュメントについて
</div>

このドキュメントの REST API 関連コンテンツは、OpenAPI 仕様に基づいて構成されています。生の yaml spec は、このドキュメントアプリケーションの公開リポジトリで確認できます。[こちら](https://github.com/Benzinga/doc-site-mintlify)をご覧ください。

OpenAPI ベースの各ページの「Playground」セクションには、REST クライアントが直接組み込まれています。API キーを取得すれば、ドキュメントポータルを離れることなく API 呼び出しをテストできます。なお、他の GUI REST クライアント ([Postman](https://www.getpostman.com/)、[Insomnia](https://insomnia.rest/)、[Paw](https://paw.cloud/)) や、cUrl1 のような CLI ツールを使って呼び出しをテストすることもできます。

また、OpenAPI 仕様は Swagger エコシステムのツールと互換性があるため、[任意のプログラミング言語向けのクライアントを生成する](http://swagger.io/swagger-codegen/)こともできます。

<Info>
  このドキュメントは AI に対応しています。Ctrl + K (PC) または Cmd + K (Mac) で質問してみてください。

<div id="acessing-the-data">
  ## データへのアクセス
</div>

<Steps>
  <Step title="アカウントにログイン">
    アカウントを作成するには、「[Get Started](https://www.benzinga.com/apis/licensing/register)」をクリックします。
  </Step>

  <Step title="APIキーをコピー">
    ユーザーダッシュボードにログインしたら、APIキーをコピーします。
  </Step>

  <Step title="構築を始めましょう！">
    認証には、URL クエリパラメータ「token」に設定したAPIキーを使用します。
  </Step>
</Steps>

<div id="best-practices">
  ## ベストプラクティス
</div>

可能な場合は、Benzinga 提供のクライアントを使用することをお勧めします。多くのエンドポイントは当社の [Python library](https://github.com/Benzinga/benzinga-python-client) で利用できます。また、公開 [Github](https://github.com/benzinga) ではサンプル実装やクライアントライブラリも提供しています。

最後に、Calendar、Signals、ニュース API の利用時には、特に履歴データの取り込みではなくリアルタイム取り込みを行う場合、差分更新の利用を強く推奨します。これは、Calendar API と Signals API では `parameters[updated]` クエリパラメータを、ニュース API では `updatedSince` クエリパラメータを使用することで実現できます。これらのパラメータを使用することで、レイテンシを最小限に抑えられます。
