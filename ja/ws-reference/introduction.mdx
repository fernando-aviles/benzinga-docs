Benzinga WebSocket API ドキュメントへようこそ。このガイドでは、Benzinga のリアルタイムデータストリームへのアクセスを開始する方法を説明します。これにより、リアルタイムの市場気配値、約定、ニュース、その他の重要なインサイトを含む金融データの継続的な更新を購読できるようになります。

<div id="websocket-endpoint">
  ## WebSocket エンドポイント
</div>

すべてのWebSocket接続は、以下のエンドポイントを使用して確立できます。

```
wss://api.benzinga.com
```

<div id="authentication">
  ## 認証
</div>

Benzinga WebSocket API では、Bearer 認証が必要です。WebSocket 接続を開始する際は、次の形式で接続 URL のクエリパラメータに APIトークンを含めてください。
`token` クエリパラメータには、`token: <token>` の形式で APIトークンを必ず指定する必要があります。

APIトークンの取得方法と管理方法の詳細については、[認証](./authentication) ページを参照してください。

<div id="subscribing-to-streams">
  ## ストリームの購読
</div>

接続後は、株価の気配値、取引データ、ニュース更新など、要件に応じて特定のデータストリームを購読できます。各サブスクリプションには、一意のIDと、データタイプに応じた特定のパラメータが必要です。

<div id="example-subscription-message">
  ### サブスクリプションメッセージの例
</div>

リアルタイムの株価ストリームを購読するには、次の形式の JSON メッセージを送信します。

```json
{
  "action": "subscribe",
  "symbol": "AAPL",
  "type": "quote"
}
```

<div id="heartbeat-mechanism">
  ## ハートビートの仕組み
</div>

接続を正常に維持し、クライアントとサーバーの同期を保つため、Benzinga WebSocket API にはハートビート機構が備わっています。

<div id="client-side-heartbeat">
  ### クライアント側ハートビート
</div>

* プレーンテキストフレームとして ping メッセージ (例: 文字列 ping) を送信すると、サーバーはプレーンテキストの pong メッセージ (例: 文字列 pong) を返します。

* これにより、接続が引き続き有効で、応答可能な状態にあることを確認できます。

<div id="example">
  #### 例
</div>

* クライアントから送信: `ping`

* サーバーが応答: `pong`

<div id="server-side-heartbeat">
  ### サーバー側ハートビート
</div>

* 接続をアクティブな状態に維持するため、サーバーは 10 秒ごとに自動でクライアントへ ping フレームを送信します。

* クライアントはこの ping フレームに応答する必要があります (ほとんどの WebSocket ライブラリでは自動的に処理されます) 。

<div id="handling-heartbeats">
  ### ハートビートの処理
</div>

* ほとんどの WebSocket クライアントライブラリ (JavaScript、Python、Go など) は、サーバーから送信される ping を自動的に処理し、pong を返します。

* ただし、独自の WebSocket クライアントを実装している場合は、接続が切断されないよう、サーバーの ping フレームに適切に応答するようにしてください。

<div id="rate-limiting">
  ## レート制限
</div>

不正利用を防止し、サービスの安定性を確保するため、Benzinga API ではレート制限を設けています。1 つの APIトークンにつき、同時に許可される接続は 1 つのみです。レート制限を超えた場合は、エラーレスポンスが返されます。
