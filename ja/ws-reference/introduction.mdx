Benzinga ウェブソケット APIドキュメントへようこそ。このガイドでは、Benzingaのリアルタイムデータストリームへのアクセスを開始する方法を説明します。これにより、ライブの市場価格、取引、ニュース、その他の重要なインサイトを含む財務データの更新を継続的に購読できるようになります。

<div id="websocket-endpoint">
  ## ウェブソケットエンドポイント
</div>

すべてのウェブソケット接続は、以下のエンドポイントで確立できます。

```
wss://api.benzinga.com
```

<div id="authentication">
  ## 認証
</div>

Benzinga ウェブソケット API では Bearer 認証が必要です。ウェブソケット接続を開始する際は、接続 URL のクエリパラメータとして APIトークンを次の形式で指定してください。
`token` クエリパラメータには、必ず `token: <token>` の形式で APIトークンを含めてください。

APIトークンの取得方法と管理方法の詳細については、[認証](./authentication) ページを参照してください。

<div id="subscribing-to-streams">
  ## ストリームの購読
</div>

接続が確立したら、株価クオート、取引データ、ニュース更新など、要件に応じて特定のデータストリームを購読できます。各サブスクリプションには、一意の識別子と、データ種別に応じた固有のパラメータが必要です。

<div id="example-subscription-message">
  ### サブスクリプションメッセージの例
</div>

リアルタイムの株価ストリームを購読するには、以下の形式のJSONメッセージを送信します。

```json
{
  "action": "subscribe",
  "symbol": "AAPL",
  "type": "quote"
}
```

<div id="heartbeat-mechanism">
  ## ハートビート機構
</div>

接続の健全性を維持し、クライアントとサーバーの同期を保つため、Benzinga ウェブソケット API にはハートビート機構が備わっています。

<div id="client-side-heartbeat">
  ### クライアント側ハートビート
</div>

* `ping` などの文字列をプレーンテキストフレームの ping メッセージとして送信すると、サーバーは `pong` などの文字列をプレーンテキストの pong メッセージとして返します。

* これにより、接続が引き続き有効で、応答可能な状態にあることを確認できます。

<div id="example">
  #### 例
</div>

* クライアントが送信: `ping`

* サーバーが応答: `pong`

<div id="server-side-heartbeat">
  ### サーバー側のハートビート
</div>

* 接続をアクティブな状態に維持するため、サーバーは 10 秒ごとにクライアントへ自動的に ping フレームを送信します。

* クライアントはこの ping フレームに応答する必要があります (ほとんどのウェブソケットライブラリでは自動的に処理されます) 。

<div id="handling-heartbeats">
  ### ハートビートの処理
</div>

* ほとんどのウェブソケットクライアントライブラリ (JavaScript、Python、Go など) では、サーバーからの ping を自動的に処理し、pong を返します。

* ただし、独自のウェブソケットクライアントを実装している場合は、接続が切断されないよう、サーバーの ping フレームに適切に応答するようにしてください。

<div id="rate-limiting">
  ## レート制限
</div>

Benzinga API では、不正利用を防ぎ、サービスの安定性を確保するためにレート制限を設けています。APIトークンごとに同時接続できるのは 1 つまでです。レート制限を超えた場合は、エラーレスポンスが返されます。
