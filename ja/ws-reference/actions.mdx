このページでは、Benzinga ウェブソケット API の操作に使用できる対応アクションを一覧で示します。これらのアクションは、セッションの健全性を維持し、直近のメッセージ履歴にアクセスするために使用します。

> **注:** `list` アクションで返されるメッセージ数は指定できません。実際の件数は、対象のキーに対して現在キャッシュされているメッセージ数によって異なります。

<div id="available-actions">
  ## 利用可能なアクション
</div>

| Action   | 説明                                                            | 例        |
| -------- | ------------------------------------------------------------- | -------- |
| `replay` | 直近100件までのメッセージを再配信します。返されるメッセージ数は、対象キーのキャッシュ状況によって異なる場合があります。 | `replay` |
| `ping`   | ハートビート信号を送信して、ウェブソケットセッションを維持します。                             | `ping`   |

<div id="example-usage">
  ## 使用例
</div>

**以下の形式で、ウェブソケットクライアントからアクションを送信します。**
ping の使用例:

```bash
ping
```

レスポンス:

```bash
pong
```

replay の使用例:

```bash
replay
```

レスポンス:

```json
{
  "id": "unique-uuid-for-response",
  "api_version": "websocket/v1",
  "kind": "stream_type",
  "data": {
    "action": "created/updated/deleted",
    "id": "insight-id",
    "content": {
      ...  
    },
    "timestamp": "2024-10-08T10:00:00Z"
  }
}
```
