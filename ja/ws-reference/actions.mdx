このページでは、Benzinga WebSocket API とのやり取りに使用できる対応アクションを一覧で示します。これらのアクションは、セッションの健全性を維持し、直近のメッセージ履歴にアクセスするのに役立ちます。

> **注:** `list` アクションで返されるメッセージ数は制御できません。実際の件数は、指定したキーに対して現在キャッシュされているメッセージ数によって決まります。

<div id="available-actions">
  ## 利用可能なアクション
</div>

| Action   | Description                                                      | Example  |
| -------- | ---------------------------------------------------------------- | -------- |
| `replay` | 直近最大100件のメッセージを再送します。返されるメッセージ数は、キーで利用可能なキャッシュの状況によって異なる場合があります。 | `replay` |
| `ping`   | ハートビート信号を送信して、WebSocket セッションを維持します。                             | `ping`   |

<div id="example-usage">
  ## 使用例
</div>

**以下の形式で、WebSocket クライアントからアクションを送信します。**
ping の使用例:

```bash
ping
```

レスポンス:

```bash
pong
```

`replay`の使用例:

```bash
replay
```

レスポンス:

```json
{
  "id": "unique-uuid-for-response",
  "api_version": "websocket/v1",
  "kind": "stream_type",
  "data": {
    "action": "created/updated/deleted",
    "id": "insight-id",
    "content": {
      ...  
    },
    "timestamp": "2024-10-08T10:00:00Z"
  }
}
```
