<div id="overview">
  ## 概要
</div>

リアルタイムの決算説明会トランスクリプトを購読すると、進行中の決算説明会での発言を文単位でリアルタイムに受信できます。リアルタイムのトランスクリプトビューアや分析ツールの構築に最適です。

<div id="key-features">
  ### 主な機能
</div>

* **ライブ文字起こし**: 決算説明会での発話内容が、文単位でリアルタイムに更新されます
* **単語レベルのタイミング情報**: 各文には、各単語のタイムスタンプと信頼度スコアが含まれます
* **話者識別**: 誰が話しているか (経営陣、アナリストなど) を把握できます
* **ステータス追跡**: トランスクリプトが進行中か完了済みかを確認できます
* **柔軟なサブスクリプション**: 特定のティッカー、または `*` を使ってすべてのトランスクリプトを購読できます
* **モックテスト**: ライブの決算説明会が行われていない場合でも、モックデータで統合をテストできます

<div id="use-cases">
  ### ユースケース
</div>

* リアルタイムの決算説明会トランスクリプトビューアを構築する
* ライブ字幕やアクセシビリティツールを作成する
* 決算説明会のリアルタイムセンチメント分析を行う
* 言及されたタイミングで重要なフレーズや指標を抽出する
* ライブコール中に特定のキーワードやトピックに対してアラートを送信する
* 正確な時刻データとともに完全なトランスクリプトをアーカイブする

<div id="quick-start">
  ## クイックスタート
</div>

上部の対話型ウェブソケットテスターを使用して接続し、リアルタイムでストリームをテストします。

<div id="connection-url">
  ### 接続 URL
</div>

```
wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### クエリパラメーター
</div>

| パラメーター  | 必須 | 説明                       |
| ------- | -- | ------------------------ |
| `token` | はい | BenzingaウェブソケットAPIのtoken |

<div id="subscription-commands">
  ## サブスクリプションコマンド
</div>

サブスクリプションを管理するには、JSONコマンドを送信します：

<div id="subscribe-to-a-ticker">
  ### 特定のティッカーを購読する
</div>

```json
{
  "action": "subscribe",
  "ticker": "AAPL"
}
```

<div id="subscribe-to-all-transcripts">
  ### すべてのトランスクリプトを購読する
</div>

```json
{
  "action": "subscribe",
  "ticker": "*"
}
```

<div id="unsubscribe-from-a-ticker">
  ### ティッカーの購読を解除する
</div>

```json
{
  "action": "unsubscribe",
  "ticker": "AAPL"
}
```

<div id="list-active-transcripts">
  ### アクティブなトランスクリプトの一覧
</div>

```json
{
  "action": "list"
}
```

<div id="keep-connection-alive">
  ### 接続を維持する
</div>

```json
{
  "action": "ping"
}
```

<div id="test-with-mock-data">
  ### モックデータでテストする
</div>

ライブのトランスクリプトが利用できない場合は、テストにモックモードを使用します。

```json
{
  "action": "subscribe",
  "ticker": "AAPL",
  "mock": true
}
```

サポートされているモックティッカー: `AAPL`, `MSFT`, `TSLA`

</div>

<div id="key-fields">
  ### 主要フィールド
</div>

| フィールド                     | 説明                                       |
| ------------------------- | ---------------------------------------- |
| `sentence`                | 発話された文の完全なテキスト                           |
| `sequence`                | トランスクリプト内でのこの文の順序 (0、1、2...)             |
| `speaker`                 | 話者の名前／識別子                                |
| `status`                  | `IN_PROGRESS` (ライブ) または `COMPLETE` (完了)  |
| `confidence`              | 全体の精度に対する信頼度 (0.0 - 1.0)                 |
| `words`                   | タイミング情報と信頼度を含む各単語の配列                     |
| `start_time` / `end_time` | 文の時刻を示す ISO 8601 タイムスタンプ                 |

<div id="transcript-status">
  ### トランスクリプトのステータス
</div>

| ステータス         | 説明                            |
| ------------- | ----------------------------- |
| `IN_PROGRESS` | トランスクリプトは現在ライブで、更新が継続的に反映されます |
| `COMPLETE`    | トランスクリプトは終了しており、以降の更新はありません   |

<div id="word-level-data">
  ## 単語単位のデータ
</div>

各文には、単語単位の詳細情報が含まれます。

```json
{
  "text": "Revenue",
  "confidence": 0.99,
  "start": 0,
  "end": 450
}
```

* **text**: 単語そのもの
* **confidence**: 認識の信頼度 (0.0 - 1.0)
* **start/end**: 文頭からのミリ秒単位のタイミングオフセット

<div id="interactive-commands">
  ## 対話型コマンド
</div>

| Command       | Parameters | Description                   |
| ------------- | ---------- | ----------------------------- |
| `subscribe`   | `ticker`   | ティッカーのトランスクリプト更新を購読します        |
| `unsubscribe` | `ticker`   | ティッカーの更新受信を停止します              |
| `list`        | -          | アクティブな/利用可能なトランスクリプトの一覧を取得します |
| `subscribed`  | -          | 現在の購読一覧を取得します                 |
| `ping`        | -          | 接続を維持します (`pong` を返します)       |
| `echo`        | `message`  | エコーテスト (送信したメッセージをそのまま返します)   |

<div id="best-practices">
  ## ベストプラクティス
</div>

<div id="real-time-display">
  ### リアルタイム表示
</div>

* **バッファ管理**: `sequence` フィールドを使用して、文を順番に表示します
* **話者の書式設定**: `speaker` に応じて表示形式を変えます (経営陣とアナリスト) 
* **ステータスインジケーター**: `IN_PROGRESS` と `COMPLETE` の状態を示す視覚的なインジケーターを表示します

<div id="performance">
  ### パフォーマンス
</div>

* **ワイルドカード使用時の注意**: `ticker: "*"` を使用すると、すべてのトランスクリプトを購読することになり、トラフィック量が非常に多くなる可能性があります
* **対象を絞った購読**: 帯域幅を抑えるため、必要なティッカーのみを購読してください
* **Heartbeat**: 接続を維持するため、30～60秒ごとに `ping` を送信してください

<div id="data-quality">
  ### データ品質
</div>

* **信頼度によるフィルタリング**: 信頼度スコアの低い文は、無視するかフラグを付けることを検討してください
* **単語精度**: 単語レベルの信頼度を使って、不確かな文字起こしを特定します
* **言語処理**: テキストが適切に表示されるよう、`language` フィールドを確認してください

<div id="testing">
  ### テスト
</div>

* **モックモード**: 実際の呼び出しを行わない開発時は `mock: true` を使用します
* **エコーテスト**: 接続とメッセージ処理を確認するには `echo` アクションを使用します
* **一覧コマンド**: 現在利用可能な項目を確認するには `list` を使用します

<div id="example-integration">
  ## 連携例
</div>

```javascript
const ws = new WebSocket(
  'wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN'
);

ws.onopen = () => {
  // Appleのトランスクリプトを購読する
  ws.send(JSON.stringify({
    action: 'subscribe',
    ticker: 'AAPL'
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  // 話者付きで文章を表示する
  console.log(`[${data.sequence}] ${data.speaker}: ${data.sentence}`);

  // トランスクリプトが完了したか確認する
  if (data.status === 'COMPLETE') {
    console.log('Transcript finished');
  }
};

// 接続を維持する
setInterval(() => {
  ws.send(JSON.stringify({ action: 'ping' }));
}, 30000);
```
