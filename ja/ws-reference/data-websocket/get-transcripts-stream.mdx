<div id="overview">
  ## 概要
</div>

リアルタイムの決算説明会トランスクリプトを購読すると、ライブの決算説明会での発言内容を文単位で受信できます。リアルタイムのトランスクリプトビューアや分析ツールの構築に最適です。

<div id="key-features">
  ### 主な機能
</div>

* **ライブ文字起こし**: 決算説明会中に話者が話すたびに、文ごとの更新をリアルタイムで取得できます
* **単語レベルのタイミング**: 各文には、各単語のタイムスタンプと信頼度スコアが含まれます
* **話者識別**: 誰が話しているか (経営陣、アナリストなど) を把握できます
* **ステータス追跡**: トランスクリプトが進行中か完了済みかを確認できます
* **柔軟なサブスクリプション**: 特定のティッカー、または `*` を使用してすべてのトランスクリプトを購読できます
* **モックテスト**: ライブの説明会が行われていない場合でも、モックデータを使って統合をテストできます

<div id="use-cases">
  ### ユースケース
</div>

* リアルタイムの決算説明会トランスクリプトビューアを構築する
* ライブ字幕表示やアクセシビリティツールを作成する
* 決算説明会のリアルタイムセンチメント分析を行う
* 言及されたタイミングで重要なフレーズや指標を抽出する
* ライブの決算説明会中に特定のキーワードやトピックに対してアラートを出す
* 正確なタイミングデータとともに完全なトランスクリプトをアーカイブする

<div id="quick-start">
  ## クイックスタート
</div>

上記の対話型WebSocketテスターを使用して接続し、リアルタイムでストリームをテストします。

<div id="connection-url">
  ### 接続URL
</div>

```
wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### クエリパラメータ
</div>

| パラメータ   | 必須 | 説明                               |
| ------- | -- | -------------------------------- |
| `token` | はい | お使いの Benzinga WebSocket API トークン |

<div id="subscription-commands">
  ## サブスクリプションコマンド
</div>

サブスクリプションを制御するには、JSONコマンドを送信します:

<div id="subscribe-to-a-ticker">
  ### 特定のティッカーを購読する
</div>

```json
{
  "action": "subscribe",
  "ticker": "AAPL"
}
```

<div id="subscribe-to-all-transcripts">
  ### すべてのトランスクリプトを購読
</div>

```json
{
  "action": "subscribe",
  "ticker": "*"
}
```

<div id="unsubscribe-from-a-ticker">
  ### ティッカーの購読解除
</div>

```json
{
  "action": "unsubscribe",
  "ticker": "AAPL"
}
```

<div id="list-active-transcripts">
  ### アクティブなトランスクリプトを一覧表示する
</div>

```json
{
  "action": "list"
}
```

<div id="keep-connection-alive">
  ### 接続を維持する
</div>

```json
{
  "action": "ping"
}
```

<div id="test-with-mock-data">
  ### モックデータでテストする
</div>

ライブのトランスクリプトを利用できない場合は、テストにモックモードを使用してください。

```json
{
  "action": "subscribe",
  "ticker": "AAPL",
  "mock": true
}
```

対応しているモックティッカー: `AAPL`, `MSFT`, `TSLA`

<div id="key-fields">
  ### 主要フィールド
</div>

| フィールド                     | 説明                                      |
| ------------------------- | --------------------------------------- |
| `sentence`                | 発話された文全体のテキスト                           |
| `sequence`                | トランスクリプト内におけるこの文の順序 (0, 1, 2...)        |
| `speaker`                 | 話者の名前/識別子                               |
| `status`                  | `IN_PROGRESS` (ライブ) または `COMPLETE` (完了) |
| `confidence`              | 全体的な精度に対する信頼度 (0.0 - 1.0)               |
| `words`                   | タイミングと信頼度を含む各単語の配列                      |
| `start_time` / `end_time` | 文のタイミングを示す ISO 8601 タイムスタンプ             |

<div id="transcript-status">
  ### トランスクリプトのステータス
</div>

| ステータス         | 説明                            |
| ------------- | ----------------------------- |
| `IN_PROGRESS` | トランスクリプトは現在ライブ配信中で、更新を受信しています |
| `COMPLETE`    | トランスクリプトは終了しており、以降の更新はありません   |

<div id="word-level-data">
  ## 単語レベルのデータ
</div>

各文には、各単語に関する詳細情報が含まれます。

```json
{
  "text": "Revenue",
  "confidence": 0.99,
  "start": 0,
  "end": 450
}
```

* **text**: 単語自体
* **confidence**: 認識信頼度 (0.0 - 1.0)
* **start/end**: 文の開始からのミリ秒単位のオフセット

<div id="interactive-commands">
  ## 対話型コマンド
</div>

| コマンド          | パラメータ     | 説明                             |
| ------------- | --------- | ------------------------------ |
| `subscribe`   | `ticker`  | 指定したティッカーのトランスクリプト更新を購読します     |
| `unsubscribe` | `ticker`  | 指定したティッカーの更新の受信を停止します          |
| `list`        | -         | アクティブまたは利用可能なトランスクリプトの一覧を取得します |
| `subscribed`  | -         | 現在購読中の一覧を取得します                 |
| `ping`        | -         | 接続を維持します (`pong` が返されます)       |
| `echo`        | `message` | エコーテスト (送信したメッセージがそのまま返されます)   |

<div id="best-practices">
  ## ベストプラクティス
</div>

<div id="real-time-display">
  ### リアルタイム表示
</div>

* **バッファ管理**: `sequence` フィールドを使用して、文を順番どおりに表示します
* **話者の書式設定**: `speaker` に応じて表示形式を変えます (経営陣とアナリスト) 
* **ステータスインジケーター**: `IN_PROGRESS` と `COMPLETE` のステータスを視覚的なインジケーターで示します

<div id="performance">
  ### パフォーマンス
</div>

* **ワイルドカード使用時の注意**: `ticker: "*"` を使用すると、すべてのトランスクリプトをサブスクライブすることになり、受信量が非常に多くなる可能性があります
* **対象を絞ったサブスクリプション**: 帯域幅を抑えるため、必要なティッカーのみをサブスクライブしてください
* **ハートビート**: 接続を維持するため、30〜60秒ごとに `ping` を送信してください

<div id="data-quality">
  ### データ品質
</div>

* **信頼度によるフィルタリング**: 信頼度スコアの低い文は、無視するかフラグ付けすることを検討してください
* **単語精度**: 単語レベルの信頼度を使って、不確かな文字起こしを特定します
* **言語処理**: 適切にテキストを表示するため、`language` フィールドを確認します

<div id="testing">
  ### テスト
</div>

* **モックモード**: ライブの決算説明会が行われていない開発時は `mock: true` を使用します
* **エコーテスト**: 接続とメッセージ処理を確認するには `echo` アクションを使用します
* **一覧コマンド**: 現在利用可能な項目を確認するには `list` を使用します

<div id="example-integration">
  ## 統合の例
</div>

```javascript
const ws = new WebSocket(
  'wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN'
);

ws.onopen = () => {
  // Appleのトランスクリプトを購読する
  ws.send(JSON.stringify({
    action: 'subscribe',
    ticker: 'AAPL'
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  // 話者付きで文章を表示する
  console.log(`[${data.sequence}] ${data.speaker}: ${data.sentence}`);

  // トランスクリプトが完了したか確認する
  if (data.status === 'COMPLETE') {
    console.log('Transcript finished');
  }
};

// 接続を維持する
setInterval(() => {
  ws.send(JSON.stringify({ action: 'ping' }));
}, 30000);
```
