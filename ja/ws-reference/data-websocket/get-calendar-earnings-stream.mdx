<div id="overview">
  ## 概要
</div>

リアルタイムの決算発表を購読すると、EPS、売上高、業績サプライズを含む企業の決算情報の更新を即座に受け取れます。

<div id="key-features">
  ### 主な機能
</div>

* **リアルタイム決算情報**: 決算発表をリアルタイムで通知します
* **包括的なデータ**: EPS、売上高、予想値、実績値、サプライズ指標を含みます
* **銘柄で絞り込み**: 特定のティッカーまたはISINを指定して購読できます
* **決算期**: Q1、Q2、Q3、Q4、および通期 (FY) の結果を追跡できます
* **重要度評価**: 市場での重要性を示す0～5のスケール

<div id="use-cases">
  ### ユースケース
</div>

* リアルタイムの決算カレンダーやアラートを構築
* 決算サプライズや予想上振れ・下振れの傾向を追跡
* ポートフォリオ内の企業の決算をリアルタイムで監視
* 決算シーズン向けのダッシュボードを作成
* 決算結果に基づく取引戦略を自動化

<div id="quick-start">
  ## クイックスタート
</div>

上のインタラクティブなウェブソケットテスターを使用して接続し、ストリームをリアルタイムでテストできます。

<div id="connection-url">
  ### 接続 URL
</div>

```
wss://api.benzinga.com/api/v2.1/calendar/earnings/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### クエリパラメータ
</div>

| パラメータ     | 必須  | 説明                                                    |
| --------- | --- | ----------------------------------------------------- |
| `token`   | はい  | お使いの Benzinga ウェブソケット API token (bz.production***)  |
| `tickers` | いいえ | ティッカーシンボルのカンマ区切りリスト (例: `AAPL,MSFT`)                  |
| `isins`   | いいえ | 結果を絞り込むための ISIN のカンマ区切りリスト                            |

<div id="key-fields">
  ### 主要フィールド
</div>

| フィールド                  | 説明                  |
| ---------------------- | ------------------- |
| `eps`                  | 実際に報告された1株当たり利益     |
| `eps_est`              | アナリストのコンセンサスEPS予想   |
| `eps_surprise`         | 実績値と予想値の差           |
| `eps_surprise_percent` | 予想値に対するサプライズの割合     |
| `revenue`              | 実際に報告された売上高         |
| `revenue_est`          | アナリストのコンセンサス売上高予想   |
| `importance`           | 市場での重要度 (0～5、5が最高)  |

<div id="message-actions">
  ### メッセージアクション
</div>

| アクション                 | 説明       |
| --------------------- | -------- |
| `created` / `Created` | 新しい決算発表  |
| `updated` / `Updated` | 決算データの更新 |
| `deleted` / `Deleted` | 発表の削除    |

<div id="interactive-commands">
  ## 対話コマンド
</div>

| コマンド     | 説明                            |
| -------- | ----------------------------- |
| `ping`   | 接続を維持します (応答は `pong`)         |
| `replay` | 直近のキャッシュ済みメッセージを最大100件まで再送します |

<div id="best-practices">
  ## ベストプラクティス
</div>

* **重複排除**: 重複処理を防ぐため、`id` フィールドを使用します
* **重要度フィルター**: 市場への影響が大きい主要な決算に注目する場合は、重要度が高い (4～5) ものを対象にします
* **サプライズアラート**: 大幅な決算サプライズに対するしきい値を設定します
* **タイミング**: `time` フィールドは EST/EDT の24時間形式を使用する点に注意してください
