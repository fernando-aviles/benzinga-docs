<div id="overview">
  ## 概要
</div>

リアルタイムの決算発表を購読し、EPS、売上高、決算サプライズを含む会社の決算結果に関する更新を即座に受け取れます。

<div id="key-features">
  ### 主な機能
</div>

* **リアルタイム決算情報**: 決算発表が行われると同時に即時通知を受け取れます
* **包括的なデータ**: EPS、売上高、予想値、実績値、サプライズ指標を含みます
* **銘柄でフィルタリング**: 特定のティッカーまたはISINを購読する
* **決算期間**: Q1、Q2、Q3、Q4、および通期 (FY) の結果を追跡
* **重要度**: 市場での重要性を示す0～5の尺度

<div id="use-cases">
  ### ユースケース
</div>

* リアルタイムの決算カレンダーとアラートを構築する
* 決算サプライズや予想上振れ／下振れの傾向を追跡する
* ポートフォリオ内の会社の決算をリアルタイムで監視する
* 決算シーズン向けのdashboardを作成する
* 決算結果に基づく取引戦略を自動化する

<div id="quick-start">
  ## クイックスタート
</div>

上部の対話型 WebSocket テスターを使って接続し、ストリームをリアルタイムでテストします。

<div id="connection-url">
  ### 接続URL
</div>

```
wss://api.benzinga.com/api/v2.1/calendar/earnings/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### クエリパラメータ
</div>

| パラメータ     | 必須  | 説明                                              |
| --------- | --- | ----------------------------------------------- |
| `token`   | はい  | Benzinga WebSocket API トークン (bz.production***)  |
| `tickers` | いいえ | ティッカーシンボルのカンマ区切りリスト (例: `AAPL,MSFT`)            |
| `isins`   | いいえ | 絞り込み用の ISIN のカンマ区切りリスト                          |

<div id="key-fields">
  ### 主要フィールド
</div>

| フィールド                  | 説明                    |
| ---------------------- | --------------------- |
| `eps`                  | 実際に報告された1株当たり利益       |
| `eps_est`              | アナリストのコンセンサスEPS予想     |
| `eps_surprise`         | 実績値と予想値の差             |
| `eps_surprise_percent` | 予想値に対するサプライズ率         |
| `revenue`              | 実際に報告された売上高           |
| `revenue_est`          | アナリストのコンセンサス売上高予想     |
| `importance`           | 市場における重要度 (0～5、5が最高)  |

<div id="message-actions">
  ### メッセージアクション
</div>

| アクション                 | 説明       |
| --------------------- | -------- |
| `created` / `Created` | 新規の決算発表  |
| `updated` / `Updated` | 決算データの更新 |
| `deleted` / `Deleted` | 発表の削除    |

<div id="interactive-commands">
  ## 対話型コマンド
</div>

| コマンド     | 説明                            |
| -------- | ----------------------------- |
| `ping`   | 接続を維持します (`pong` が返されます)      |
| `replay` | 直近のキャッシュ済みメッセージを最大100件まで再送します |

<div id="best-practices">
  ## ベストプラクティス
</div>

* **重複排除**: 重複処理を防ぐために `id` フィールドを使用します
* **重要度フィルター**: 市場への影響が大きい主要な決算に絞るには、重要度の高い (4-5) イベントに注目します
* **サプライズアラート**: 決算サプライズが大きい場合に備えて、しきい値を設定します
* **タイミング**: `time` フィールドは EST/EDT の 24 時間表記である点に注意してください
