<div id="overview">
  ## 概要
</div>

証券に関する強気・弱気両面の分析をリアルタイムで購読できます。株式について、強気シナリオと弱気シナリオの両方から、バランスの取れた見方を得られます。

<div id="key-features">
  ### 主な機能
</div>

* **バランスの取れた分析**: 市場を総合的に把握できるよう、強気・弱気の両方のケースを受け取れます
* **リアルタイム更新**: ケースが作成、更新、または削除されると、即座に通知を受け取れます
* **銘柄でフィルタリング**: 特定のティッカーまたはISINを指定して購読できます
* **市場センチメント**: 楽観的な見方と悲観的な見方の両方を把握できます

<div id="use-cases">
  ### ユースケース
</div>

* 議論の両面を可視化するセンチメント分析ダッシュボードを構築する
* 強気・弱気センチメントの変化を時系列で追跡する
* 市場認識の大きな変化をユーザーに通知する
* バランスの取れた投資リサーチを顧客に提供する

<div id="quick-start">
  ## クイックスタート
</div>

上記のインタラクティブなウェブソケットテスターを使用して接続し、ストリームをリアルタイムでテストできます。

<div id="connection-url">
  ### 接続 URL
</div>

```
wss://api.benzinga.com/api/v1/bulls_bears_say/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### クエリパラメータ
</div>

| パラメータ     | 必須  | 説明                                                 |
| --------- | --- | -------------------------------------------------- |
| `token`   | はい  | Benzinga ウェブソケット API の token (bz.production***)  |
| `tickers` | いいえ | ティッカーのカンマ区切りリスト (例: `AAPL,TSLA`)                   |
| `isins`   | いいえ | 絞り込みに使用する ISIN のカンマ区切りリスト                          |

<div id="message-example">
  ## メッセージ例
</div>

```json
{
  "id": "websocket123",
  "api_version": "websocket/v1",
  "kind": "bull_bear_case",
  "data": {
    "action": "created",
    "id": "case123",
    "timestamp": "2024-10-01T10:00:00Z",
    "content": {
      "bull_case": "Strong earnings growth and market expansion expected.",
      "bear_case": "Increased competition and regulatory pressures.",
      "ticker": "AAPL",
      "updated": 1696153200
    }
  }
}
```

<div id="message-actions">
  ### メッセージアクション
</div>

| アクション                 | 説明             |
| --------------------- | -------------- |
| `created` / `Created` | 新しい強気/弱気ケースの公開 |
| `updated` / `Updated` | 既存ケースの更新       |
| `deleted` / `Deleted` | ケースの削除         |

<div id="interactive-commands">
  ## 対話型コマンド
</div>

| コマンド     | 説明                          |
| -------- | --------------------------- |
| `ping`   | 接続を維持します (`pong` が返されます)    |
| `replay` | 直近のキャッシュメッセージを最大100件まで再送します |

<div id="best-practices">
  ## ベストプラクティス
</div>

* **重複排除**: 重複メッセージを処理しないように、`id` フィールドを使用します
* **ハートビート**: 30～60秒ごとに `ping` コマンドを送信します
* **バランスの取れた提示**: 公平な分析を行うため、強気・弱気の両方の見方を表示します
* **更新の追跡**: 直近の変更を特定するため、`updated` タイムスタンプを監視します
