<div id="overview">
  ## 概要
</div>

証券に関する強気・弱気のケース分析をリアルタイムで購読できます。株式について、強気 (上昇) シナリオと弱気 (下落) シナリオの両面から、バランスの取れた見解を取得できます。

<div id="key-features">
  ### 主な機能
</div>

* **バランスの取れた分析**: 包括的な市場インサイトのために、強気・弱気の両方のケースを取得できます
* **リアルタイム更新**: ケースの作成、更新、削除時に即座に通知を受け取れます
* **証券で絞り込み**: 特定のティッカーまたはISINを指定してサブスクライブできます
* **市場センチメント**: 楽観論と悲観論の両方の見方を把握できます

<div id="use-cases">
  ### ユースケース
</div>

* 議論の両面を示すセンチメント分析dashboardを構築する
* 強気/弱気センチメントの変化を時系列で追跡する
* 市場の見方の大きな変化をユーザーに通知する
* 顧客にバランスの取れた投資リサーチを提供する

<div id="quick-start">
  ## クイックスタート
</div>

上記のインタラクティブなWebSocketテスターを使用して接続し、ストリームをリアルタイムでテストします。

<div id="connection-url">
  ### 接続URL
</div>

```
wss://api.benzinga.com/api/v1/bulls_bears_say/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### クエリパラメータ
</div>

| パラメータ     | 必須  | 説明                                                 |
| --------- | --- | -------------------------------------------------- |
| `token`   | はい  | Benzinga WebSocket API の token (bz.production***)  |
| `tickers` | いいえ | ティッカーシンボルのカンマ区切りリスト (例: `AAPL,TSLA`)               |
| `isins`   | いいえ | 絞り込みに使用する ISIN のカンマ区切りリスト                          |

<div id="message-example">
  ## メッセージ例
</div>

```json
{
  "id": "websocket123",
  "api_version": "websocket/v1",
  "kind": "bull_bear_case",
  "data": {
    "action": "created",
    "id": "case123",
    "timestamp": "2024-10-01T10:00:00Z",
    "content": {
      "bull_case": "Strong earnings growth and market expansion expected.",
      "bear_case": "Increased competition and regulatory pressures.",
      "ticker": "AAPL",
      "updated": 1696153200
    }
  }
}
```

<div id="message-actions">
  ### メッセージアクション
</div>

| アクション                 | 説明                  |
| --------------------- | ------------------- |
| `created` / `Created` | 新しい強気/弱気ケースが公開されました |
| `updated` / `Updated` | 既存のケースが更新されました      |
| `deleted` / `Deleted` | ケースが削除されました         |

<div id="interactive-commands">
  ## 対話型コマンド
</div>

| コマンド     | 説明                            |
| -------- | ----------------------------- |
| `ping`   | 接続を維持します (`pong` で応答)         |
| `replay` | 直近のキャッシュ済みメッセージを最大100件まで再送します |

<div id="best-practices">
  ## ベストプラクティス
</div>

* **重複排除**: 重複メッセージを処理しないように `id` フィールドを使用します
* **ハートビート**: 30〜60秒ごとに `ping` コマンドを送信します
* **バランスの取れた提示**: 公平な分析になるよう、強気ケースと弱気ケースの両方を表示します
* **更新の追跡**: `updated` タイムスタンプを監視して最新の変更を把握します
