---
title: "エラー"
description: "エラー発生時に Benzinga API がどのように応答するか。"
---

Benzinga では、API リクエストの成功または失敗を示すために、一般的な HTTP レスポンスコードを使用します。
一般に、2xx の範囲のコードは成功を示します。
4xx の範囲のコードは、提供された情報に問題があるために発生したエラーを示します (例: 必須パラメータ
が省略されている、課金に失敗した、など) 。
5xx の範囲のコードは、Benzinga のサーバーで発生したエラーを示します。

プログラムで処理可能な 4xx エラーの一部には、
報告されたエラーの内容を簡潔に説明するエラーコードが含まれます。

<div id="parameters">
  ### パラメータ
</div>

<ResponseField name="type" type="string">
  返されるエラーの種別です。

  <code>invalid&#95;request&#95;error</code>、<code>not&#95;found&#95;error</code>、<code>creation&#95;error</code>、
  <code>update&#95;error</code>、<code>deletion&#95;error</code>、<code>forbidden&#95;error</code>、または<code>authentication&#95;error</code>のいずれかです。
</ResponseField>

<ResponseField name="message" type="string">
  エラーの詳細を示す、人が読めるメッセージです。
</ResponseField>

<ResponseField name="params" type="array">
  エラーが特定のパラメータに関連する場合、無効なパラメータの一覧が含まれます。
</ResponseField>
