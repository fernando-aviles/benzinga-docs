Le damos la bienvenida a la documentación de la API de Benzinga. Esta guía le ayudará a comprender cómo interactuar con la API de Benzinga para crear,
consultar, actualizar y eliminar recursos de Benzinga mediante solicitudes HTTP.

<div id="base-url">
  ## URL base
</div>

Todas las URL a las que se hace referencia en la documentación tienen la siguiente base:

```
https://api.benzinga.com
```

<div id="authentication">
  ## Autenticación
</div>

La API de Benzinga utiliza autenticación de portador. Al realizar solicitudes, debe incluir su token de API en el parámetro de consulta `token`
con el formato `token: <token>`. Para obtener más información, consulte la página de [Autenticación](./authentication).

<div id="errors">
  ## Errores
</div>

Cuando se produce un error, la API de Benzinga responde con un código de estado HTTP estándar y un objeto JSON que contiene más
detalles sobre el error. Para obtener más información, consulta la página de [Errores](./errors).

<div id="rate-limiting">
  ## Límites de tasa
</div>

Tenga en cuenta que la API de Benzinga tiene límites de tasa para evitar abusos y garantizar la estabilidad del servicio. Si supera estos límites,
sus solicitudes se restringirán y recibirá una respuesta `429 Too Many Requests`.

***

Esperamos que esta guía le ayude a empezar a usar la API de Benzinga. Si tiene alguna pregunta, no dude en ponerse en contacto
con nuestro equipo de soporte.
