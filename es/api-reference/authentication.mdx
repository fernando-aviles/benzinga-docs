La API de Benzinga utiliza **claves de API** para autenticar las solicitudes. Tu clave de API es un identificador único que otorga acceso a datos y funcionalidades específicas según tu suscripción.

> [!IMPORTANT]
> **Mantén tu clave de API segura.** No la compartas en lugares de acceso público, como GitHub, código del lado del cliente o comunicaciones no seguras. Si crees que tu clave se ha visto comprometida, ponte en contacto con soporte de inmediato.

Puedes ver y administrar tu clave de API en tu [Benzinga Developer Console](https://www.api.benzinga.com/token).

<div id="authentication-methods">
  ## Métodos de autenticación
</div>

La API de Benzinga admite dos métodos de autenticación. Recomendamos encarecidamente usar el método de **encabezado HTTP** en aplicaciones de producción, ya que es más seguro y evita que su clave aparezca en los registros de URL.

<div id="1-http-header-recommended">
  ### 1. Encabezado HTTP (recomendado)
</div>

Para autenticarse mediante el encabezado, incluya el encabezado `Authorization` con el valor `token <YOUR_API_KEY>`.

```http
Authorization: token <YOUR_API_KEY>
```

<div id="2-query-parameter">
  ### 2. Parámetro de consulta
</div>

Para pruebas rápidas o cuando no sea posible modificar el encabezado, puede enviar su clave como un parámetro de consulta llamado `token`.

```http
https://api.benzinga.com/api/v2/news?token=<YOUR_API_KEY>
```

<div id="code-examples">
  ## Ejemplos de código
</div>

A continuación, se muestran ejemplos listos para producción para conectarse a la API de Benzinga en lenguajes habituales.

<CodeGroup>
  ```bash cURL
  # Recomendado: autenticación mediante encabezado
  curl -L 'https://api.benzinga.com/api/v2/news?pageSize=1' \
  -H 'Authorization: token YOUR_API_KEY' \
  -H 'Accept: application/json'

  # Alternativa: parámetro de consulta
  curl -L 'https://api.benzinga.com/api/v2/news?pageSize=1&token=YOUR_API_KEY'
  ```

  ```python Python
  import requests

  url = "https://api.benzinga.com/api/v2/news"

  # Recomendado: autenticación mediante encabezado
  headers = {
      "Authorization": "token YOUR_API_KEY",
      "Accept": "application/json"
  }

  params = {
      "pageSize": 1
  }

  try:
      response = requests.get(url, headers=headers, params=params)
      response.raise_for_status() # Genera un error detallado para respuestas 4xx/5xx
      data = response.json()
      print(data)
  except requests.exceptions.RequestException as e:
      print(f"Error al obtener los datos: {e}")
  ```

  ```javascript Node.js
  const axios = require('axios');

  const url = 'https://api.benzinga.com/api/v2/news';
  const apiKey = 'YOUR_API_KEY';

  async function getNews() {
    try {
      const response = await axios.get(url, {
        headers: {
          'Authorization': `token ${apiKey}`,
          'Accept': 'application/json'
        },
        params: {
          pageSize: 1
        }
      });
      
      console.log(response.data);
    } catch (error) {
      if (error.response) {
        // La solicitud se realizó y el servidor respondió con un código de estado
        // fuera del rango 2xx
        console.error('Estado del error:', error.response.status);
        console.error('Datos del error:', error.response.data);
      } else {
        console.error('Mensaje de error:', error.message);
      }
    }
  }

  getNews();
  ```

  ```go Go
  package main

  import (
  	"fmt"
  	"io/ioutil"
  	"net/http"
  )

  func main() {
  	url := "https://api.benzinga.com/api/v2/news?pageSize=1"
  	apiKey := "YOUR_API_KEY"

  	req, err := http.NewRequest("GET", url, nil)
  	if err != nil {
  		panic(err)
  	}

  	// Recomendado: autenticación mediante encabezado
  	req.Header.Add("Authorization", fmt.Sprintf("token %s", apiKey))
  	req.Header.Add("Accept", "application/json")

  	client := &http.Client{}
  	resp, err := client.Do(req)
  	if err != nil {
  		panic(err)
  	}
  	defer resp.Body.Close()

  	if resp.StatusCode >= 400 {
  		fmt.Printf("Error: código de estado %d\n", resp.StatusCode)
  		return
  	}

  	body, err := ioutil.ReadAll(resp.Body)
  	if err != nil {
  		panic(err)
  	}

  	fmt.Println(string(body))
  }
  ```

<div id="troubleshooting">
  ## Solución de problemas
</div>

Errores comunes de autenticación y cómo resolverlos.

| Código de estado | Mensaje        | Posible causa          | Resolución                                                                                                                                                             |
| :--------------- | :------------- | :--------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **401**          | `Unauthorized` | Clave de API no válida | Verifique que su clave de API sea correcta y que no se haya regenerado. Asegúrese de que no se hayan copiado espacios de más.                                          |
| **401**          | `Unauthorized` | Falta la clave de API  | Asegúrese de que el encabezado `Authorization` tenga el formato correcto: `token <KEY>`, o de que esté presente el parámetro `token`.                                  |
| **403**          | `Forbidden`    | Permisos insuficientes | Su clave de API es válida, pero su plan no incluye acceso al punto final solicitado. Póngase en contacto con el equipo de ventas o de soporte para actualizar su plan. |
