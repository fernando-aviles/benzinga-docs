<div id="overview">
  ## Descripción general
</div>

Use este punto final para activar una entrega de prueba del webhook a su punto final configurado. Esto le permite verificar que su integración de webhook funcione correctamente antes de recibir datos de producción en tiempo real.

<div id="testing-your-integration">
  ## Prueba de la integración
</div>

Cuando hagas una llamada a este punto final, Benzinga enviará una carga útil de webhook de prueba a la URL `destination` que hayas configurado. Esta entrega de prueba sigue el mismo formato y la misma lógica de reintentos que las entregas de webhooks en producción.

<div id="what-to-expect">
  ### Qué esperar
</div>

1. **Respuesta inmediata**: La API devuelve un código de estado `200` si la entrega de prueba se activó correctamente
2. **Carga útil de prueba**: El punto final del webhook recibe una carga útil de prueba con el mismo formato que los datos de producción
3. **Encabezados de entrega**: La entrega de prueba incluye el encabezado `X-BZ-Delivery`, igual que las entregas de producción

<div id="verify-your-integration">
  ### Verifique su integración
</div>

Use este punto final para comprobar:

* Su punto final de webhook es accesible públicamente
* Su punto final puede procesar correctamente el formato de carga útil del webhook
* Su punto final responde con los códigos de estado adecuados (2xx en caso de éxito)
* Su punto final responde dentro del tiempo de espera de 30 segundos
* Su lógica de idempotencia gestiona correctamente el encabezado `X-BZ-Delivery` y el campo `id` de la carga útil

<div id="best-practices">
  ## Mejores prácticas
</div>

* Pruebe primero con puntos finales de webhook que no sean de producción
* Verifique que su punto final responda con los códigos de estado `200` o `204`
* Confirme que sus registros y su monitoreo capturen la entrega de prueba
* Compruebe que su lógica de deduplicación funcione con el ID de la entrega de prueba
* Pruebe escenarios de error devolviendo temporalmente códigos de estado de error

<div id="troubleshooting">
  ## Solución de problemas
</div>

<div id="424-delivery-error">
  ### 424 Error de entrega
</div>

Si recibes un código de estado `424`, el sistema no pudo entregar la carga útil de prueba a tu punto final de destino. Causas comunes:

* La URL de destino no es accesible públicamente
* El punto final de destino devuelve códigos de estado de error
* Problemas de conectividad de red
* Errores en el certificado SSL/TLS del punto final de destino

<div id="400-invalid-request">
  ### 400 Solicitud no válida
</div>

Verifica que se proporcionen todos los parámetros obligatorios y que tengan el formato correcto:

* `destination` debe ser una URL HTTPS válida
* `version` debe ser `webhook/v1`
* `kind` debe ser `News/v1`
