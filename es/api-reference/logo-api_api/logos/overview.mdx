<div id="introduction">
  ## Introducción
</div>

La API de logos de Benzinga proporciona acceso a logos de alta calidad de empresas, fondos y criptomonedas. Puede buscar logos mediante varios tipos de identificadores, incluidos symbols, números CIK, códigos ISIN, CUSIP, FIGI y más.

<div id="available-endpoints">
  ## Endpoints disponibles
</div>

<div id="logossearch">
  ### `/logos/search`
</div>

Busca logotipos mediante varios identificadores. Devuelve las URL de los logotipos, detalles de los valores y metadatos.

<div id="logossync">
  ### `/logos/sync`
</div>

Endpoint de sincronización masiva para obtener varios logotipos con soporte de paginación.

<div id="search-filters">
  ## Filtros de búsqueda
</div>

La API permite realizar búsquedas flexibles mediante dos parámetros principales:

* `search_keys` - El valor del identificador que desea buscar
* `search_keys_type` - El tipo de identificador (opcional; el valor predeterminado es `symbol`)

<div id="1-filter-by-symbol">
  ### 1. Filtrar por símbolo
</div>

Busque logotipos usando symbols de acciones.

**Parámetros:**

* `search_keys`: Símbolo (p. ej., `AAPL`, `TSLA`, `MSFT`)
* `search_keys_type`: `symbol` (opcional; este es el valor predeterminado)

**Ejemplo:**

```
GET /api/v2/logos/search?search_keys=AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="2-filter-by-cik">
  ### 2. Filtrar por CIK
</div>

Busque mediante el Central Index Key (CIK) de la SEC. Admite valores con y sin ceros a la izquierda.

**Parámetros:**

* `search_keys`: número de CIK (p. ej., `320193` o `0000320193`)
* `search_keys_type`: `cik`

**Ejemplo:**

```
GET /api/v2/logos/search?search_keys=320193&search_keys_type=cik&fields=logo_light,logo_dark
```

**Ejemplo con CIK con ceros a la izquierda:**

```
GET /api/v2/logos/search?search_keys=0000320193&search_keys_type=cik&fields=logo_light,logo_dark
```

<div id="3-filter-by-isin">
  ### 3. Filtrar por ISIN
</div>

Busque usando el Número Internacional de Identificación de Valores (ISIN).

**Parámetros:**

* `search_keys`: código ISIN (p. ej., `US0378331005`)
* `search_keys_type`: `isin`

**Ejemplo:**

```
GET /api/v2/logos/search?search_keys=US0378331005&search_keys_type=isin&fields=logo_light,logo_dark
```

<div id="4-filter-by-figi-share-class">
  ### 4. Filtrar por clase de acción FIGI
</div>

Busque usando el Identificador Global de Instrumentos Financieros de Bloomberg para clases de acciones.

**Parámetros:**

* `search_keys`: ID de la clase de acción FIGI (p. ej., `BBG001SQKGD7`)
* `search_keys_type`: `figi_share_class`

**Ejemplo:**

```
GET /api/v2/logos/search?search_keys=BBG001SQKGD7&search_keys_type=figi_share_class&fields=logo_light,logo_dark
```

<div id="5-filter-by-cusip">
  ### 5. Filtrar por CUSIP
</div>

Busque usando el número CUSIP.

**Parámetros:**

* `search_keys`: número CUSIP (p. ej., `88160R101`)
* `search_keys_type`: `cusip`

**Ejemplo:**

```
GET /api/v2/logos/search?search_keys=88160R101&search_keys_type=cusip&fields=logo_light,logo_dark
```

<div id="6-filter-by-symbol-with-currency-country">
  ### 6. Filtrar por símbolo con moneda y país
</div>

Especifique los symbols con códigos de país o moneda usando dos puntos (`:`) o punto (`.`) como delimitador.

**Parámetros:**

* `search_keys`: Símbolo con calificador (p. ej., `AAPL:US`, `AAPL:USD`, `AAPL.US`)
* `search_keys_type`: `symbol`

**Ejemplos:**

```
GET /api/v2/logos/search?search_keys=AAPL:US&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL:USD&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=AAPL.US&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="7-filter-by-exchange-and-symbol">
  ### 7. Filtrar por bolsa y símbolo
</div>

Busque combinando el código de la bolsa con el símbolo. Admite formatos tanto de estilo Bloomberg (heredado) como de estilo LSEG.

**Correspondencia de estilo Bloomberg (heredado):**

* `search_keys`: Bolsa y símbolo (p. ej., `NASDAQ:AAPL`)
* `search_keys_type`: `symbol`

**Código LSEG de 3 caracteres (actual):**

* `search_keys`: Código de bolsa y símbolo (p. ej., `NSQ:AAPL`)
* `search_keys_type`: `symbol`

**Ejemplos:**

```
GET /api/v2/logos/search?search_keys=NASDAQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=NSQ:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="8-filter-by-iso-mic-and-symbol">
  ### 8. Filtrar por MIC ISO y símbolo
</div>

Busque usando el código ISO de identificación de mercado combinado con el símbolo.

**Parámetros:**

* `search_keys`: código MIC y símbolo (p. ej., `XYNS:AAPL`)
* `search_keys_type`: `symbol`

**Ejemplo:**

```
GET /api/v2/logos/search?search_keys=XYNS:AAPL&search_keys_type=symbol&fields=logo_light,logo_dark
```

<div id="9-crypto-logo-filters">
  ### 9. Filtros de logos de criptomonedas
</div>

Busque logos de criptomonedas en varios formatos. La API reconoce automáticamente los formatos de criptomonedas incluso cuando no se especifica `search_keys_type`.

**Formatos admitidos:**

* `CRYPTO:BTC`
* `CRYPTO/BTC`
* `$BTC`
* `BTC/USD`

**Ejemplos:**

```
GET /api/v2/logos/search?search_keys=CRYPTO:BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=CRYPTO/BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=$BTC&fields=logo_light,logo_dark
GET /api/v2/logos/search?search_keys=BTC/USD&fields=logo_light,logo_dark
```

<div id="available-fields">
  ## Campos disponibles
</div>

Puede solicitar tipos de logotipo específicos con el parámetro `fields`. Los campos disponibles incluyen:

* `logo_light` - Logotipo para tema claro (PNG)
* `logo_dark` - Logotipo para tema oscuro (PNG)
* `logo_vector_light` - Logotipo vectorial para tema claro (SVG)
* `logo_vector_dark` - Logotipo vectorial para tema oscuro (SVG)
* `mark_light` - Marca/icono para tema claro (PNG)
* `mark_dark` - Marca/icono para tema oscuro (PNG)
* `mark_vector_light` - Marca vectorial para tema claro (SVG)
* `mark_vector_dark` - Marca vectorial para tema oscuro (SVG)
* `mark_composite_light` - Marca compuesta para tema claro (PNG)
* `mark_composite_dark` - Marca compuesta para tema oscuro (PNG)
* `mark_vector_composite_light` - Marca compuesta vectorial para tema claro (SVG)
* `mark_vector_composite_dark` - Marca compuesta vectorial para tema oscuro (SVG)

**Ejemplo de solicitud de varios campos:**

```
GET /api/v2/logos/search?search_keys=TSLA&fields=logo_light,logo_dark,mark_vector_composite_light
```

<div id="additional-parameters">
  ## Parámetros adicionales
</div>

<div id="image-customization">
  ### Personalización de imágenes
</div>

* `scale` - Escala de la imagen (p. ej., `100x100`, `300x300`)
* `max_width` - Ancho máximo de la imagen devuelta
* `composite_radius` - Radio del borde de las imágenes compuestas (entero, intervalo `0-50`)
* `composite_auto` - Genera imágenes compuestas automáticamente (booleano)

<div id="pagination-bulk-sync-only">
  ### Paginación (solo para bulk-sync)
</div>

* `page` - Número de página (entero)
* `pagesize` - Número de resultados por página (entero)
* `updated_since` - Filtrar por marca temporal de actualización (formato ISO 8601)

<div id="security-details">
  ### Detalles de los valores
</div>

* `securities` - Incluye información detallada de los valores en la respuesta (boolean)

<div id="authentication">
  ## Autenticación
</div>

Todos los endpoints de la API de Logos requieren autenticación mediante una clave de API. Incluya su token como parámetro de consulta:

```
?token=YOUR_API_KEY
```

<div id="best-practices">
  ## Mejores prácticas
</div>

1. **Especifique search&#95;keys&#95;type** - Aunque la API usa `symbol` de forma predeterminada, especificar explícitamente el tipo mejora la claridad y el rendimiento
2. **Solicite solo los campos necesarios** - Solicite únicamente los tipos de logo que necesite para minimizar el tamaño de la respuesta y mejorar el rendimiento
3. **Use bulk-sync para conjuntos de datos grandes** - Al recuperar varios logos, use el endpoint `/logos/bulk-sync` con paginación
4. **Almacene en caché las respuestas** - Las URL de los logos incluyen marcas de tiempo de expiración; aplique el almacenamiento en caché según corresponda para reducir las llamadas a la API
5. **Gestione los errores correctamente** - Verifique siempre el campo `ok` en las respuestas y gestione adecuadamente los casos de error
