Esta página enumera las acciones admitidas que puede usar para interactuar con la API WebSocket de Benzinga. Estas acciones ayudan a mantener el estado de la sesión y a acceder al historial reciente de mensajes.

> **Nota:** No puede controlar la cantidad de mensajes devueltos por la acción `list`. El número real depende de cuántos mensajes estén almacenados actualmente en caché para una clave determinada.

<div id="available-actions">
  ## Acciones disponibles
</div>

| Acción   | Descripción                                                                                                                            | Ejemplo  |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------- | -------- |
| `replay` | Reenvía hasta los últimos 100 mensajes. La cantidad de mensajes devueltos puede variar según la disponibilidad en caché para la clave. | `replay` |
| `ping`   | Mantiene activa la sesión de WebSocket enviando una señal de latido.                                                                   | `ping`   |

<div id="example-usage">
  ## Ejemplo de uso
</div>

**Envíe una acción a través de su cliente WebSocket con el siguiente formato:**
Ejemplo de uso de ping:

```bash
ping
```

Respuesta:

```bash
pong
```

Ejemplo de uso de replay:

```bash
replay
```

Respuesta:

```json
{
  "id": "unique-uuid-for-response",
  "api_version": "websocket/v1",
  "kind": "stream_type",
  "data": {
    "action": "created/updated/deleted",
    "id": "insight-id",
    "content": {
      ...  
    },
    "timestamp": "2024-10-08T10:00:00Z"
  }
}
```
