---
title: "Errores"
description: "Cómo responde la API de Benzinga cuando se producen errores."
---

Benzinga usa los códigos de respuesta HTTP convencionales para indicar el éxito o el fracaso de una solicitud a la API.
En general: Los códigos del rango 2xx indican éxito.
Los códigos del rango 4xx indican un error que hizo fallar la solicitud debido a la información proporcionada (p. ej., se omitió un parámetro obligatorio,
falló un cargo, etc.).
Los códigos del rango 5xx indican un error en los servidores de Benzinga.

Algunos errores 4xx que podrían manejarse mediante programación incluyen
un código de error que explica brevemente el error reportado.

<div id="parameters">
  ### Parámetros
</div>

<ResponseField name="type" type="string">
  El tipo de error devuelto.

  Uno de <code>invalid&#95;request&#95;error</code>, <code>not&#95;found&#95;error</code>, <code>creation&#95;error</code>,
  <code>update&#95;error</code>, <code>deletion&#95;error</code>, <code>forbidden&#95;error</code> o <code>authentication&#95;error</code>.
</ResponseField>

<ResponseField name="message" type="string">
  Un mensaje legible para las personas que proporciona más detalles sobre el error.
</ResponseField>

<ResponseField name="params" type="array">
  Si el error es específico de un parámetro, contendrá una lista de los parámetros no válidos.
</ResponseField>
