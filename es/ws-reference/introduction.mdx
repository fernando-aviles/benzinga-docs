Bienvenido a la documentación de la API WebSocket de Benzinga. Esta guía le ayudará a empezar a acceder a los flujos de datos en tiempo real de Benzinga, lo que le permitirá suscribirse a actualizaciones continuas de datos financieros, incluidas cotizaciones de mercado en tiempo real, operaciones, noticias y otros datos clave.

<div id="websocket-endpoint">
  ## Punto final de WebSocket
</div>

Todas las conexiones WebSocket pueden establecerse mediante el siguiente punto final:

```
wss://api.benzinga.com
```

<div id="authentication">
  ## Autenticación
</div>

La API WebSocket de Benzinga requiere autenticación de portador. Al iniciar una conexión WebSocket, incluya su token de API en la URL de conexión como parámetro de consulta con el siguiente formato:
debe incluir su token de API en el parámetro de consulta `token` con el formato `token: <token>`.

Consulte la página de [Autenticación](./authentication) para obtener más información sobre cómo obtener y administrar su token de API.

<div id="subscribing-to-streams">
  ## Suscripción a flujos
</div>

Una vez conectado, puede suscribirse a flujos de datos específicos según sus necesidades, como cotizaciones de acciones, datos de operaciones o noticias. Cada suscripción requiere un identificador único y parámetros específicos en función del tipo de datos.

<div id="example-subscription-message">
  ### Ejemplo de mensaje de suscripción
</div>

Para suscribirte a un flujo de cotizaciones bursátiles en tiempo real, envía un mensaje JSON con el siguiente formato:

```json
{
  "action": "subscribe",
  "symbol": "AAPL",
  "type": "quote"
}
```

<div id="heartbeat-mechanism">
  ## Mecanismo de heartbeat
</div>

Para mantener una conexión estable y garantizar que el cliente y el servidor permanezcan sincronizados, la API WebSocket de Benzinga incluye un mecanismo de latido.

<div id="client-side-heartbeat">
  ### Heartbeat del cliente
</div>

* Si envía un mensaje ping como una trama de texto sin formato (p. ej., la cadena ping), el servidor responderá con un mensaje pong también como texto sin formato (p. ej., la cadena pong).

* Esto puede usarse para verificar que la conexión siga activa y respondiendo.

<div id="example">
  #### Ejemplo
</div>

* El cliente envía: `ping`

* El servidor responde: `pong`

<div id="server-side-heartbeat">
  ### Heartbeat del servidor
</div>

* El servidor enviará automáticamente una trama de ping al cliente cada 10 segundos para garantizar que la conexión permanezca activa.

* Se espera que los clientes respondan a esta trama de ping (la mayoría de las bibliotecas de WebSocket lo gestionan automáticamente).

<div id="handling-heartbeats">
  ### Gestión de heartbeats
</div>

* La mayoría de las bibliotecas cliente de WebSocket (p. ej., las de JavaScript, Python o Go) gestionan automáticamente los pings entrantes del servidor y responden con un pong.

* Sin embargo, si está implementando un cliente WebSocket personalizado, asegúrese de que responda correctamente a las tramas de ping del servidor para evitar que se cierre la conexión.

<div id="rate-limiting">
  ## Límites de tasa
</div>

Ten en cuenta que la API de Benzinga tiene límites de tasa para evitar abusos y garantizar la estabilidad del servicio. Solo se permite 1 conexión por token de API a la vez. Si superas el límite de tasa, recibirás una respuesta de error.
