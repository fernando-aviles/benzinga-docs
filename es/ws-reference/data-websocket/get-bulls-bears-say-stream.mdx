<div id="overview">
  ## Descripción general
</div>

Suscríbase a análisis en tiempo real de escenarios alcistas y bajistas para valores. Obtenga perspectivas equilibradas sobre escenarios positivos (alcistas) y negativos (bajistas) para acciones.

<div id="key-features">
  ### Funcionalidades clave
</div>

* **Análisis equilibrado**: Reciba tanto escenarios alcistas como bajistas para obtener una visión completa del mercado
* **Actualizaciones en tiempo real**: Reciba notificaciones instantáneas cuando se creen, se actualicen o se eliminen casos
* **Filtrar por valor**: Suscríbase a tickers o ISIN específicos
* **Sentimiento del mercado**: Comprenda tanto los puntos de vista optimistas como los pesimistas

<div id="use-cases">
  ### Casos de uso
</div>

* Crear dashboards de análisis de sentimiento que muestren ambas posturas
* Realizar un seguimiento de los cambios en el sentimiento alcista/bajista a lo largo del tiempo
* Alertar a los usuarios sobre cambios significativos en la percepción del mercado
* Proporcionar a los clientes análisis de inversión equilibrados

<div id="quick-start">
  ## Inicio rápido
</div>

Usa el probador interactivo de WebSocket que aparece arriba para conectarte y probar el flujo en tiempo real.

<div id="connection-url">
  ### URL de conexión
</div>

```
wss://api.benzinga.com/api/v1/bulls_bears_say/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### Parámetros de consulta
</div>

| Parámetro | Obligatorio | Descripción                                                       |
| --------- | ----------- | ----------------------------------------------------------------- |
| `token`   | Sí          | Tu token de la API WebSocket de Benzinga (bz.production***)       |
| `tickers` | No          | Lista de símbolos ticker separada por comas (p. ej., `AAPL,TSLA`) |
| `isins`   | No          | Lista de ISIN separados por comas para filtrar                    |

<div id="message-example">
  ## Ejemplo de mensaje
</div>

```json
{
  "id": "websocket123",
  "api_version": "websocket/v1",
  "kind": "bull_bear_case",
  "data": {
    "action": "created",
    "id": "case123",
    "timestamp": "2024-10-01T10:00:00Z",
    "content": {
      "bull_case": "Strong earnings growth and market expansion expected.",
      "bear_case": "Increased competition and regulatory pressures.",
      "ticker": "AAPL",
      "updated": 1696153200
    }
  }
}
```

<div id="message-actions">
  ### Acciones del mensaje
</div>

| Acción                | Descripción                          |
| --------------------- | ------------------------------------ |
| `created` / `Created` | Nuevo caso alcista/bajista publicado |
| `updated` / `Updated` | Caso existente modificado            |
| `deleted` / `Deleted` | Caso eliminado                       |

<div id="interactive-commands">
  ## Comandos interactivos
</div>

| Comando  | Descripción                                                 |
| -------- | ----------------------------------------------------------- |
| `ping`   | Mantiene activa la conexión (responde con `pong`)           |
| `replay` | Reenvía hasta los últimos 100 mensajes almacenados en caché |

<div id="best-practices">
  ## Prácticas recomendadas
</div>

* **Eliminación de duplicados**: Use el campo `id` para evitar procesar mensajes duplicados
* **Heartbeat**: Envíe comandos `ping` cada 30-60 segundos
* **Presentación equilibrada**: Muestre tanto los casos alcistas como los bajistas para ofrecer un análisis equilibrado
* **Seguimiento de actualizaciones**: Supervise la marca de tiempo `updated` para identificar cambios recientes
