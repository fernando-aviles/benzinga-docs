<div id="overview">
  ## Descripción general
</div>

Suscríbase a las transcripciones en tiempo real de las llamadas de resultados y reciba actualizaciones frase por frase a medida que se van pronunciando durante las llamadas en vivo. Ideal para crear visualizadores de transcripciones y herramientas de análisis en tiempo real.

<div id="key-features">
  ### Características clave
</div>

* **Transcripción en vivo**: Obtén actualizaciones de frases en tiempo real a medida que los participantes hablan durante las llamadas de resultados
* **Marcas de tiempo por palabra**: Cada frase incluye marcas de tiempo de cada palabra y puntuaciones de confianza
* **Identificación de hablantes**: Haz seguimiento de quién está hablando (ejecutivos, analistas, etc.)
* **Seguimiento del estado**: Supervisa si las transcripciones están en curso o se han completado
* **Suscripción flexible**: Suscríbete a tickers específicos o a todas las transcripciones con `*`
* **Pruebas simuladas**: Prueba tu integración con datos simulados cuando no haya llamadas en vivo activas

<div id="use-cases">
  ### Casos de uso
</div>

* Crear visualizadores de transcripciones de llamadas de resultados en tiempo real
* Crear herramientas de subtitulado en directo y accesibilidad
* Realizar análisis de sentimiento en tiempo real de llamadas de resultados
* Extraer frases clave y métricas a medida que se mencionan
* Generar alertas sobre palabras clave o temas específicos durante llamadas en directo
* Archivar transcripciones completas con marcas de tiempo precisas

<div id="quick-start">
  ## Inicio rápido
</div>

Usa el probador interactivo de WebSocket que aparece arriba para conectarte y probar el stream en tiempo real.

<div id="connection-url">
  ### URL de conexión
</div>

```
wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### Parámetros de consulta
</div>

| Parámetro | Obligatorio | Descripción                                 |
| --------- | ----------- | ------------------------------------------- |
| `token`   | Sí          | Tu token de la API de WebSocket de Benzinga |

<div id="subscription-commands">
  ## Comandos de suscripción
</div>

Envíe comandos JSON para gestionar sus suscripciones:

<div id="subscribe-to-a-ticker">
  ### Suscribirse a un ticker
</div>

```json
{
  "action": "subscribe",
  "ticker": "AAPL"
}
```

<div id="subscribe-to-all-transcripts">
  ### Suscribirse a todas las transcripciones
</div>

```json
{
  "action": "subscribe",
  "ticker": "*"
}
```

<div id="unsubscribe-from-a-ticker">
  ### Cancelar la suscripción a un ticker
</div>

```json
{
  "action": "unsubscribe",
  "ticker": "AAPL"
}
```

<div id="list-active-transcripts">
  ### Listar transcripciones activas
</div>

```json
{
  "action": "list"
}
```

<div id="keep-connection-alive">
  ### Mantener la conexión abierta
</div>

```json
{
  "action": "ping"
}
```

<div id="test-with-mock-data">
  ### Prueba con datos simulados
</div>

Cuando no haya transcripciones en vivo disponibles, use el modo simulado para hacer pruebas:

```json
{
  "action": "subscribe",
  "ticker": "AAPL",
  "mock": true
}
```

Tickers de prueba admitidos: `AAPL`, `MSFT`, `TSLA`

<div id="key-fields">
  ### Campos clave
</div>

| Campo                     | Descripción                                                  |
| ------------------------- | ------------------------------------------------------------ |
| `sentence`                | Texto completo de la frase pronunciada                       |
| `sequence`                | Orden de esta frase en la transcripción (0, 1, 2...)         |
| `speaker`                 | Nombre/identificador de la persona que habla                 |
| `status`                  | `IN_PROGRESS` (en curso) o `COMPLETE` (finalizado)           |
| `confidence`              | Confianza general en la precisión (0.0 - 1.0)                |
| `words`                   | Array de palabras individuales con hora y nivel de confianza |
| `start_time` / `end_time` | Marcas de tiempo ISO 8601 para la sincronización de la frase |

<div id="transcript-status">
  ### Estado de la transcripción
</div>

| Estado        | Descripción                                                       |
| ------------- | ----------------------------------------------------------------- |
| `IN_PROGRESS` | La transcripción está activa y recibiendo actualizaciones         |
| `COMPLETE`    | La transcripción ha finalizado; no se esperan más actualizaciones |

<div id="word-level-data">
  ## Datos por palabra
</div>

Cada oración incluye información detallada de cada palabra:

```json
{
  "text": "Revenue",
  "confidence": 0.99,
  "start": 0,
  "end": 450
}
```

* **text**: La palabra en sí
* **confidence**: Nivel de confianza del reconocimiento (0.0 - 1.0)
* **start/end**: Desplazamientos temporales en milisegundos desde el inicio de la oración

<div id="interactive-commands">
  ## Comandos interactivos
</div>

| Comando       | Parámetros | Descripción                                                       |
| ------------- | ---------- | ----------------------------------------------------------------- |
| `subscribe`   | `ticker`   | Suscribirse a las actualizaciones de transcripciones de un ticker |
| `unsubscribe` | `ticker`   | Dejar de recibir actualizaciones de un ticker                     |
| `list`        | -          | Obtener la lista de transcripciones activas o disponibles         |
| `subscribed`  | -          | Obtener la lista de tus suscripciones actuales                    |
| `ping`        | -          | Mantener la conexión activa (responde con `pong`)                 |
| `echo`        | `message`  | Prueba de eco (responde con tu mensaje)                           |

<div id="best-practices">
  ## Buenas prácticas
</div>

<div id="real-time-display">
  ### Visualización en tiempo real
</div>

* **Gestión del búfer**: Muestra las frases en orden secuencial usando el campo `sequence`
* **Formato del hablante**: Aplica un formato distinto según `speaker` (ejecutivos frente a analistas)
* **Indicadores de estado**: Muestra indicadores visuales para los estados `IN_PROGRESS` y `COMPLETE`

<div id="performance">
  ### Rendimiento
</div>

* **Precaución con comodines**: Usar `ticker: "*"` lo suscribe a TODAS las transcripciones, lo que puede generar un alto volumen
* **Suscripciones específicas**: Suscríbase solo a los tickers que necesite para reducir el ancho de banda
* **Heartbeat**: Envíe `ping` cada 30-60 segundos para mantener la conexión

<div id="data-quality">
  ### Calidad de los datos
</div>

* **Filtrado por nivel de confianza**: Considere ignorar o marcar las oraciones con puntuaciones de confianza bajas
* **Precisión a nivel de palabra**: Use la confianza a nivel de palabra para identificar transcripciones dudosas
* **Gestión del idioma**: Verifique el campo `language` para que el texto se represente correctamente

<div id="testing">
  ### Pruebas
</div>

* **Modo simulado**: Usa `mock: true` durante el desarrollo cuando no haya llamadas en vivo activas
* **Prueba de eco**: Usa la acción `echo` para verificar la conexión y la gestión de mensajes
* **Comando `list`**: Revisa `list` para ver qué está disponible actualmente

<div id="example-integration">
  ## Ejemplo de integración
</div>

```javascript
const ws = new WebSocket(
  'wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN'
);

ws.onopen = () => {
  // Suscribirse a las transcripciones de Apple
  ws.send(JSON.stringify({
    action: 'subscribe',
    ticker: 'AAPL'
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  // Mostrar la oración con el hablante
  console.log(`[${data.sequence}] ${data.speaker}: ${data.sentence}`);

  // Verificar si la transcripción está completa
  if (data.status === 'COMPLETE') {
    console.log('Transcript finished');
  }
};

// Mantener la conexión activa
setInterval(() => {
  ws.send(JSON.stringify({ action: 'ping' }));
}, 30000);
```
