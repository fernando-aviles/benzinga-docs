<div id="overview">
  ## Descripción general
</div>

Suscríbase a las transcripciones en tiempo real de las llamadas de resultados y reciba actualizaciones frase por frase a medida que se dicen durante las conferencias en vivo. Ideal para crear visores de transcripciones en tiempo real y herramientas de análisis.

<div id="key-features">
  ### Funcionalidades clave
</div>

* **Transcripción en vivo**: Obtén actualizaciones de frases en tiempo real a medida que los participantes hablan durante las llamadas de resultados
* **Marcas de tiempo a nivel de palabra**: Cada frase incluye marcas de tiempo para cada palabra y puntuaciones de confianza
* **Identificación de hablantes**: Haz seguimiento de quién está hablando (ejecutivos, analistas, etc.)
* **Seguimiento de estado**: Supervisa si las transcripciones están en curso o completadas
* **Suscripción flexible**: Suscríbete a tíkers específicos o a todas las transcripciones con `*`
* **Pruebas con datos simulados**: Prueba tu integración con datos simulados cuando no haya conferencias en vivo activas

<div id="use-cases">
  ### Casos de uso
</div>

* Cree visores de transcripciones en tiempo real de llamadas de resultados
* Cree herramientas de subtitulado en vivo y accesibilidad
* Realice análisis de sentimiento en tiempo real de llamadas de resultados
* Extraiga frases clave y métricas a medida que se mencionan
* Genere alertas sobre palabras clave o temas específicos durante llamadas en vivo
* Archive transcripciones completas con marcas de tiempo precisas

<div id="quick-start">
  ## Inicio rápido
</div>

Usa el probador interactivo de WebSocket de arriba para conectarte y probar la transmisión en tiempo real.

<div id="connection-url">
  ### URL de conexión
</div>

```
wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### Parámetros de consulta
</div>

| Parámetro | Obligatorio | Descripción                           |
| --------- | ----------- | ------------------------------------- |
| `token`   | Sí          | Token de la API WebSocket de Benzinga |

<div id="subscription-commands">
  ## Comandos de suscripción
</div>

Envía comandos JSON para gestionar tus suscripciones:

<div id="subscribe-to-a-ticker">
  ### Suscribirse a un tíker
</div>

```json
{
  "action": "subscribe",
  "ticker": "AAPL"
}
```

<div id="subscribe-to-all-transcripts">
  ### Suscribirse a todas las transcripciones
</div>

```json
{
  "action": "subscribe",
  "ticker": "*"
}
```

<div id="unsubscribe-from-a-ticker">
  ### Cancelar la suscripción a un tíker
</div>

```json
{
  "action": "unsubscribe",
  "ticker": "AAPL"
}
```

<div id="list-active-transcripts">
  ### Listar las transcripciones activas
</div>

```json
{
  "action": "list"
}
```

<div id="keep-connection-alive">
  ### Mantener la conexión activa
</div>

```json
{
  "action": "ping"
}
```

<div id="test-with-mock-data">
  ### Pruebas con datos simulados
</div>

Si no hay transcripciones en vivo disponibles, use el modo simulado para hacer pruebas:

```json
{
  "action": "subscribe",
  "ticker": "AAPL",
  "mock": true
}
```

Tíkers simulados admitidos: `AAPL`, `MSFT`, `TSLA`

<div id="key-fields">
  ### Campos clave
</div>

| Campo                     | Descripción                                                                |
| ------------------------- | -------------------------------------------------------------------------- |
| `sentence`                | Texto completo de la frase pronunciada                                     |
| `sequence`                | Orden de esta frase en la transcripción (0, 1, 2...)                       |
| `speaker`                 | Nombre/identificador de la persona que habla                               |
| `status`                  | `IN_PROGRESS` (en curso) o `COMPLETE` (completado)                         |
| `confidence`              | Nivel general de confianza en la precisión (0.0 - 1.0)                     |
| `words`                   | Arreglo de palabras individuales con marcas de tiempo y nivel de confianza |
| `start_time` / `end_time` | Marcas de tiempo ISO 8601 para el inicio y fin de la frase                 |

<div id="transcript-status">
  ### Estado de la transcripción
</div>

| Estado        | Descripción                                                       |
| ------------- | ----------------------------------------------------------------- |
| `IN_PROGRESS` | La transcripción está en curso y recibiendo actualizaciones       |
| `COMPLETE`    | La transcripción ha finalizado; no se esperan más actualizaciones |

<div id="word-level-data">
  ## Datos por palabra
</div>

Cada oración incluye información detallada de cada palabra:

```json
{
  "text": "Revenue",
  "confidence": 0.99,
  "start": 0,
  "end": 450
}
```

* **text**: La palabra en sí
* **confidence**: Confianza de reconocimiento (0.0 - 1.0)
* **inicio/fin**: Desplazamientos temporales en milisegundos desde el inicio de la oración

<div id="interactive-commands">
  ## Comandos interactivos
</div>

| Comando       | Parámetros | Descripción                                                      |
| ------------- | ---------- | ---------------------------------------------------------------- |
| `subscribe`   | `ticker`   | Suscribirse a las actualizaciones de transcripciones de un tíker |
| `unsubscribe` | `ticker`   | Dejar de recibir actualizaciones de un tíker                     |
| `list`        | -          | Obtener la lista de transcripciones activas/disponibles          |
| `subscribed`  | -          | Obtener la lista de suscripciones actuales                       |
| `ping`        | -          | Mantener la conexión activa (responde con `pong`)                |
| `echo`        | `message`  | Prueba de eco (responde con su mensaje)                          |

<div id="best-practices">
  ## Mejores prácticas
</div>

<div id="real-time-display">
  ### Visualización en tiempo real
</div>

* **Gestión del búfer**: Muestra las frases en orden de secuencia usando el campo `sequence`
* **Formato del hablante**: Aplica un formato diferente según `speaker` (ejecutivos frente a analistas)
* **Indicadores de estado**: Muestra indicadores visuales para el estado `IN_PROGRESS` frente a `COMPLETE`

<div id="performance">
  ### Rendimiento
</div>

* **Precaución con comodines**: Usar `ticker: "*"` te suscribe a TODAS las transcripciones, lo que puede suponer un volumen muy alto
* **Suscripciones específicas**: Suscríbete solo a los tíkers que necesites para reducir el ancho de banda
* **Heartbeat**: Envía `ping` cada 30-60 segundos para mantener la conexión

<div id="data-quality">
  ### Calidad de los datos
</div>

* **Filtrado por confianza**: Considere ignorar o marcar las oraciones con puntuaciones de confianza bajas
* **Precisión por palabra**: Use la confianza a nivel de palabra para identificar transcripciones dudosas
* **Gestión del idioma**: Verifique el campo `language` para asegurar la correcta visualización del texto

<div id="testing">
  ### Pruebas
</div>

* **Modo simulado**: Usa `mock: true` durante el desarrollo cuando no haya llamadas en vivo activas
* **Prueba de eco**: Usa la acción `echo` para verificar la conexión y la gestión de mensajes
* **Comando `list`**: Consulta `list` para ver qué está disponible actualmente

<div id="example-integration">
  ## Ejemplo de integración
</div>

```javascript
const ws = new WebSocket(
  'wss://api.benzinga.com/api/v1/transcripts/stream?token=YOUR_TOKEN'
);

ws.onopen = () => {
  // Suscribirse a las transcripciones de Apple
  ws.send(JSON.stringify({
    action: 'subscribe',
    ticker: 'AAPL'
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  // Mostrar la oración con el hablante
  console.log(`[${data.sequence}] ${data.speaker}: ${data.sentence}`);

  // Verificar si la transcripción está completa
  if (data.status === 'COMPLETE') {
    console.log('Transcript finished');
  }
};

// Mantener la conexión activa
setInterval(() => {
  ws.send(JSON.stringify({ action: 'ping' }));
}, 30000);
```
