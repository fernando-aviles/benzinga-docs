<div id="overview">
  ## Descripción general
</div>

Suscríbase a información de analistas en tiempo real y reciba actualizaciones sobre calificaciones de analistas, precios objetivo y recomendaciones detalladas en el momento en que se publiquen.

<div id="key-features">
  ### Características clave
</div>

* **Actualizaciones en tiempo real**: Recibe notificaciones instantáneas cuando los analistas publican nueva información o actualizan calificaciones existentes
* **Filtrado por valor**: Suscríbete a tickers o ISIN específicos para recibir solo los datos relevantes
* **Datos completos**: Incluye detalles de la firma de análisis, calificaciones, precios objetivo e información detallada
* **Seguimiento de cambios**: Supervisa cuándo se crea, actualiza o elimina información

<div id="use-cases">
  ### Casos de uso
</div>

* Hacer seguimiento de los cambios en el sentimiento de los analistas sobre los activos de la cartera
* Crear alertas en tiempo real para mejoras o rebajas de calificación
* Supervisar los cambios en los precios objetivo de varios analistas
* Consolidar información de consenso de varias firmas de análisis

<div id="quick-start">
  ## Inicio rápido
</div>

Utiliza la herramienta interactiva de prueba de WebSocket de arriba para conectarte y probar el flujo en tiempo real.

<div id="connection-url">
  ### URL de conexión
</div>

```
wss://api.benzinga.com/api/v1/analyst/insights/stream?token=YOUR_TOKEN
```

<div id="query-parameters">
  ### Parámetros de consulta
</div>

| Parámetro | Obligatorio | Descripción                                                 |
| --------- | ----------- | ----------------------------------------------------------- |
| `token`   | Sí          | Su token de la API WebSocket de Benzinga (bz.production***) |
| `tickers` | No          | Lista de tickers separados por comas (p. ej., `AAPL,MSFT`)  |
| `isins`   | No          | Lista de ISIN separados por comas para filtrar              |

</div>

```json
{
```

<div id="message-actions">
  ### Acciones del mensaje
</div>

| Acción                | Descripción                         |
| --------------------- | ----------------------------------- |
| `created` / `Created` | Nueva opinión de analista publicada |
| `updated` / `Updated` | Opinión existente modificada        |
| `deleted` / `Deleted` | Opinión eliminada                   |

<div id="interactive-commands">
  ## Comandos interactivos
</div>

| Comando  | Descripción                                                  |
| -------- | ------------------------------------------------------------ |
| `ping`   | Mantiene activa la conexión (responde con `pong`)            |
| `replay` | Recupera hasta los últimos 100 mensajes almacenados en caché |

<div id="best-practices">
  ## Mejores prácticas
</div>

* **Deduplicación**: Use el campo `id` para evitar procesar mensajes duplicados
* **Heartbeat**: Envíe comandos `ping` cada 30-60 segundos para mantener la conexión
* **Manejo de errores**: Implemente una lógica de reconexión con espera exponencial
* **Filtrado**: Use filtros de ticker/ISIN para reducir el ancho de banda
