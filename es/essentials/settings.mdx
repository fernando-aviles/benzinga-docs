---
title: 'Configuración global'
description: 'Mintlify te brinda control total sobre la apariencia y el funcionamiento de tu documentación mediante el archivo docs.json'
icon: 'gear'
---

Todos los sitios de Mintlify necesitan un archivo `docs.json` con la configuración principal. Obtén más información sobre las [propiedades](#properties) a continuación.

<div id="properties">
  ## Propiedades
</div>

<ResponseField name="name" type="string" required>
  Nombre de tu proyecto. Se usa para el título global.

  Ejemplo: `mintlify`
</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array de grupos con todas las páginas dentro de cada grupo

  <Expandable title="Navegación">
    <ResponseField name="group" type="string">
      El nombre del grupo.

      Ejemplo: `Settings`
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      Las rutas relativas a los archivos Markdown que servirán como páginas.

      Ejemplo: `["customization", "page"]`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ruta a la imagen del logotipo o a un objeto con las rutas a las imágenes del logotipo para los modos &quot;light&quot; y &quot;dark&quot;

  <Expandable title="Logotipo">
    <ResponseField name="light" type="string">
      Ruta al logotipo en modo claro
    </ResponseField>

    <ResponseField name="dark" type="string">
      Ruta al logotipo en modo oscuro
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      Destino al que enlaza el logotipo al hacer clic
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ruta a la imagen del favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de color hexadecimales para tu tema global

  <Expandable title="Colores">
    <ResponseField name="primary" type="string" required>
      El color principal. Se usa con mayor frecuencia para contenido resaltado,
      encabezados de sección y acentos, en modo claro
    </ResponseField>

    <ResponseField name="light" type="string">
      El color principal para el modo oscuro. Se usa con mayor frecuencia para contenido resaltado,
      encabezados de sección y acentos, en modo oscuro
    </ResponseField>

    <ResponseField name="dark" type="string">
      El color principal para botones importantes
    </ResponseField>

    <ResponseField name="background" type="object">
      El color de fondo tanto en modo claro como en modo oscuro

      <Expandable title="Objeto">
        <ResponseField name="light" type="string" required>
          El código de color hexadecimal del fondo en modo claro
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          El código de color hexadecimal del fondo en modo oscuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de `name` y `url` de los enlaces que quieres incluir en la barra superior

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      El nombre del botón.

      Ejemplo: `Contact us`
    </ResponseField>

    <ResponseField name="url" type="string">
      La URL a la que dirige el botón al hacer clic. Ejemplo: `https://mintlify.com/contact`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Llamada a la acción de la barra superior">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      Link muestra un botón. GitHub muestra la información del repositorio en la URL proporcionada, incluido el número de estrellas de GitHub.
    </ResponseField>

    <ResponseField name="url" type="string">
      Si es `link`: destino del botón.

      Si es `github`: enlace al repositorio del que se cargará la información de GitHub.
    </ResponseField>

    <ResponseField name="name" type="string">
      Texto dentro del botón. Solo es obligatorio si `type` es `link`.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nombres de versiones. Usa esto solo si quieres mostrar distintas versiones
  de la documentación con un menú desplegable en la barra de navegación.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Una lista de anclas que incluye `icon`, `color` y `url`.

  <Expandable title="Ancla">
    <ResponseField name="icon" type="string">
      El icono de [Font Awesome](https://fontawesome.com/search?s=brands%2Cduotone) que se usa para representar el ancla.

      Ejemplo: `comments`
    </ResponseField>

    <ResponseField name="name" type="string">
      El nombre de la etiqueta del ancla.

      Ejemplo: `Community`
    </ResponseField>

    <ResponseField name="url" type="string">
      El inicio de la URL que indica qué páginas pertenecen al ancla. Por lo general, es el nombre de la carpeta en la que colocas tus páginas.
    </ResponseField>

    <ResponseField name="color" type="string">
      El color hexadecimal del fondo del icono del ancla. También puede ser un degradado si pasas un objeto con las propiedades `from` y `to`, cada una con un color hexadecimal.
    </ResponseField>

    <ResponseField name="version" type="string">
      Se usa si quieres ocultar un ancla hasta que se seleccione la versión correcta de la documentación.
    </ResponseField>

    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Pasa `true` si quieres ocultar el ancla hasta que alguien acceda directamente mediante un enlace a documentación dentro de ella.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; o &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Sobrescribe la configuración predeterminada del ancla situada más arriba.

  <Expandable title="Objeto">
    <ResponseField name="name" default="Documentation" type="string">
      El nombre del ancla situada más arriba
    </ResponseField>

    <ResponseField name="icon" default="book-open" type="string">
      Icono de Font Awesome.
    </ResponseField>

    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: &quot;brands&quot;, &quot;duotone&quot;, &quot;light&quot;, &quot;sharp-solid&quot;, &quot;solid&quot; o &quot;thin&quot;
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Una lista de pestañas de navegación.

  <Expandable title="Pestañas">
    <ResponseField name="name" type="string">
      El nombre de la etiqueta de la pestaña.
    </ResponseField>

    <ResponseField name="url" type="string">
      El inicio de la URL que indica qué páginas pertenecen a la pestaña. Por lo general, este
      es el nombre de la carpeta en la que colocas tus páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuración de la API. Obtén más información sobre las páginas de API en [Componentes de API](/es/api-playground/demo).

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      La URL base para todos los puntos finales de la API. Si `baseUrl` es un arreglo, habilita varias
      opciones de URL base entre las que el usuario puede alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Autenticación">
        <ResponseField name="method" type="&#x22;bearer&#x22; | &#x22;basic&#x22; | &#x22;key&#x22;">
          La estrategia de autenticación que se usa en todos los puntos finales de la API.
        </ResponseField>

        <ResponseField name="name" type="string">
          El nombre del parámetro de autenticación utilizado en el playground de API.

          Si el método es `basic`, el formato debe ser `[usernameName]:[passwordName]`
        </ResponseField>

        <ResponseField name="inputPrefix" type="string">
          El valor predeterminado pensado para usarse como prefijo del campo de entrada de autenticación.

          P. ej., si `inputPrefix` es `AuthKey`, el valor predeterminado del campo de autenticación será `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuración del playground de API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="&#x22;show&#x22; | &#x22;simple&#x22; | &#x22;hide&#x22;">
          Indica si el playground se muestra, se oculta o si solo muestra el punto final sin interactividad adicional para el usuario `simple`

          Obtén más información en las [guías del playground](/es/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Habilitar esta marca garantiza que el orden de las claves en las páginas de OpenAPI coincida con el orden definido en el archivo OpenAPI.

      <Warning>Este comportamiento pronto se habilitará de forma predeterminada, momento en el que este campo quedará obsoleto.</Warning>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una cadena o un arreglo de cadenas de URL o rutas relativas que apuntan a tu
  archivo OpenAPI.

  Ejemplos:

  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```

    ```json Relative
    "openapi": "/openapi.json"
    ```

    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>
</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objeto de cuentas de redes sociales donde cada par clave-valor representa la plataforma de redes sociales y la URL de la cuenta.

  Ejemplo:

  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```

  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      Uno de los siguientes valores `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      Ejemplo: `x`
    </ResponseField>

    <ResponseField name="property" type="string">
      La URL de la plataforma social.

      Ejemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configuración para habilitar botones de comentarios

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      Habilita un botón para que los usuarios sugieran ediciones mediante pull requests
    </ResponseField>

    <ResponseField name="raiseIssue" type="boolean" default="false">
      Habilita un botón para que los usuarios reporten un problema en la documentación
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personaliza el selector del modo oscuro.

  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Indica si quieres mostrar siempre el modo claro u oscuro a los usuarios nuevos. Si no se
      establece, se usa de forma predeterminada el mismo modo que el sistema operativo del usuario.
    </ResponseField>

    <ResponseField name="isHidden" type="boolean" default="false">
      Establece `true` para ocultar el selector de modo claro/oscuro. Puedes combinar `isHidden` con `default` para forzar que tu documentación use solo el modo claro u oscuro. Por ejemplo:

      <CodeGroup>
        ```json Solo modo oscuro
        "modeToggle": {
          "default": "dark",
          "isHidden": true
        }
        ```

        ```json Solo modo claro
        "modeToggle": {
          "default": "light",
          "isHidden": true
        }
        ```
      </CodeGroup>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Una imagen de fondo que se mostrará detrás de cada página. Consulta un ejemplo con
  [Infisical](https://infisical.com/docs) y [FRPC](https://frpc.io).
</ResponseField>
