<div id="sdk-installation">
  ## Instalación del SDK
</div>

<div id="getting-started">
  ### Primeros pasos
</div>

Se requiere JDK 11 o una versión posterior.

Los ejemplos a continuación muestran cómo se utiliza un artefacto publicado del SDK:

Gradle:

```groovy
implementation 'org.benzinga:BZClient:0.2.7'
```

Maven:

```xml
<dependency>
    <groupId>org.benzinga</groupId>
    <artifactId>BZClient</artifactId>
    <version>0.2.7</version>
</dependency>
```

<div id="how-to-build">
  ### Cómo compilar
</div>

Después de clonar el repositorio de Git en tu sistema de archivos, puedes compilar el artefacto del SDK a partir del código fuente en el directorio `build` ejecutando `./gradlew build` en sistemas *nix o `gradlew.bat` en sistemas Windows.

Si deseas compilar desde el código fuente y publicar el artefacto del SDK en tu repositorio local de Maven (en tu sistema de archivos), utiliza el siguiente comando (después de clonar localmente el repositorio de Git):

En *nix:

```bash
./gradlew publishToMavenLocal -Pskip.signing
```

En Windows:

```bash
gradlew.bat publishToMavenLocal -Pskip.signing
```

</div>

| Nombre       | Tipo   | Esquema   |

| `apiKeyAuth` | apiKey | clave API |

```java
package hello.world;

            // manejar la respuesta
        }

```

<div id="retries">
  ## Reintentos
</div>

Algunos de los endpoints de este SDK admiten reintentos. Si usa el SDK sin ninguna configuración, se utilizará la estrategia de reintento predeterminada proporcionada por la API. Sin embargo, la estrategia de reintento predeterminada puede reemplazarse para cada operación o en todo el SDK.

Para cambiar la estrategia de reintento predeterminada para una sola llamada a la API, puede proporcionar un objeto `RetryConfig` mediante el método builder `retryConfig`:

```java
package hello.world;

import java.lang.Exception;
import java.util.concurrent.TimeUnit;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.operations.GetAnalystReportsRawTextDataResponse;
import org.benzinga.BZClient.utils.BackoffStrategy;
import org.benzinga.BZClient.utils.RetryConfig;

public class Application {

    public static void main(String[] args) throws Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .retryConfig(RetryConfig.builder()
                    .backoff(BackoffStrategy.builder()
                        .initialInterval(1L, TimeUnit.MILLISECONDS)
                        .maxInterval(50L, TimeUnit.MILLISECONDS)
                        .maxElapsedTime(1000L, TimeUnit.MILLISECONDS)
                        .baseFactor(1.1)
                        .jitterFactor(0.15)
                        .retryConnectError(false)
                        .build())
                    .build())
                .page(700347L)
                .pagesize(558834L)
                .call();

        if (res.modelsAnalystReportRawTexts().isPresent()) {
            // manejar respuesta
        }
    }
}
```

Si quieres sobrescribir la estrategia de reintento predeterminada para todas las operaciones que admiten reintentos, puedes proporcionar una configuración al inicializar el SDK:

```java
package hello.world;

import java.lang.Exception;
import java.util.concurrent.TimeUnit;
import org.benzinga.BZClient.Bzclient;

    public static void main(String[] args) throws Exception {

                .retryConfig(RetryConfig.builder()

                        .retryConnectError(false)
                        .build())
                    .build())
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .page(700347L)
                .pagesize(558834L)

}
```

<div id="error-handling">
  ## Manejo de errores
</div>

El manejo de errores en este SDK debería ajustarse, en líneas generales, a lo esperado. Todas las operaciones devuelven un objeto de respuesta o lanzan una excepción.

De forma predeterminada, un error de la API lanzará una excepción `models/errors/APIException`. Cuando se especifican respuestas de error personalizadas para una operación, el SDK también puede lanzar la excepción asociada. Puede consultar las tablas respectivas de *Errores* en la documentación del SDK para obtener más información sobre los posibles tipos de excepción de cada operación. Por ejemplo, el método `get` lanza las siguientes excepciones:

| Tipo de error                  | Código de estado | Tipo de contenido |
| ------------------------------ | ---------------- | ----------------- |
| models/errors/ApiErrorResponse | 400, 500         | application/json  |
| models/errors/APIException     | 4XX, 5XX         | */*               |

### Ejemplo

```java
package hello.world;

import java.lang.Exception;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.errors.ApiErrorResponse;
import org.benzinga.BZClient.models.operations.GetAnalystInsightsV1Request;
import org.benzinga.BZClient.models.operations.GetAnalystInsightsV1Response;

public class Application {

    public static void main(String[] args) throws ApiErrorResponse, Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystInsightsV1Request req = GetAnalystInsightsV1Request.builder()
                .build();

        GetAnalystInsightsV1Response res = sdk.analystInsights().get()
                .request(req)
                .call();

        if (res.modelsAnalystInsightsJSON().isPresent()) {
            // manejar respuesta
        }
    }
}
```

<div id="select-server-by-index">

</div>

Puede sobrescribir globalmente el servidor predeterminado mediante el método builder `.serverIndex(int serverIdx)` al inicializar la instancia del cliente SDK. El servidor seleccionado se usará después como predeterminado en las operaciones que lo utilicen. Esta tabla enumera los índices asociados a los servidores disponibles:

| # | Servidor                            |
| - | ----------------------------------- |
| 0 | `https://api.benzinga.com`          |
| 1 | `https://api.benzinga.com/api/v1`   |
| 2 | `https://api.benzinga.com/api/v2`   |
| 3 | `https://api.benzinga.com/api/v2.1` |
| 4 | `https://api.benzinga.com/api/v2.2` |
