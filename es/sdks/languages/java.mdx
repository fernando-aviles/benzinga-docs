<div id="sdk-installation">
  ## Instalación del SDK
</div>

<div id="getting-started">
  ### Primeros pasos
</div>

Se requiere JDK 11 o una versión posterior.

Los ejemplos a continuación muestran cómo se utiliza un artefacto publicado del SDK:

Gradle:

```groovy
implementation 'org.benzinga:BZClient:0.2.7'
```

Maven:

```xml
<dependency>
    <groupId>org.benzinga</groupId>
    <artifactId>BZClient</artifactId>
    <version>0.2.7</version>
</dependency>
```

<div id="how-to-build">
  ### Cómo compilar
</div>

Después de clonar el repositorio de git en tu sistema de archivos, puedes compilar el artefacto del SDK a partir del código fuente en el directorio `build` ejecutando `./gradlew build` en sistemas *nix o `gradlew.bat` en sistemas Windows.

Si deseas compilar desde el código fuente y publicar el artefacto del SDK en tu repositorio local de Maven (en tu sistema de archivos), usa el siguiente comando (después de clonar localmente el repositorio de git):

En *nix:

```bash
./gradlew publishToMavenLocal -Pskip.signing
```

En Windows:

```bash
gradlew.bat publishToMavenLocal -Pskip.signing
```

| Nombre       | Tipo   | Esquema      |


<div id="retries">
  ## Reintentos
</div>

Algunos de los puntos finales de este SDK admiten reintentos. Si usa el SDK sin ninguna configuración, se usará la estrategia de reintentos predeterminada que proporciona la API. Sin embargo, la estrategia de reintentos predeterminada puede sustituirse para una operación concreta o para todo el SDK.

Para cambiar la estrategia de reintentos predeterminada para una sola llamada a la API, puede proporcionar un objeto `RetryConfig` mediante el método builder `retryConfig`:

```java
package hello.world;

import java.lang.Exception;
import java.util.concurrent.TimeUnit;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.operations.GetAnalystReportsRawTextDataResponse;
import org.benzinga.BZClient.utils.BackoffStrategy;
import org.benzinga.BZClient.utils.RetryConfig;

public class Application {

    public static void main(String[] args) throws Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystReportsRawTextDataResponse res = sdk.analystReportsRawText().get()
                .retryConfig(RetryConfig.builder()
                    .backoff(BackoffStrategy.builder()
                        .initialInterval(1L, TimeUnit.MILLISECONDS)
                        .maxInterval(50L, TimeUnit.MILLISECONDS)
                        .maxElapsedTime(1000L, TimeUnit.MILLISECONDS)
                        .baseFactor(1.1)
                        .jitterFactor(0.15)
                        .retryConnectError(false)
                        .build())
                    .build())
                .page(700347L)
                .pagesize(558834L)
                .call();

        if (res.modelsAnalystReportRawTexts().isPresent()) {
            // manejar respuesta
        }
    }
}
```

Si quieres reemplazar la estrategia de reintento predeterminada para todas las operaciones compatibles con reintentos, puedes proporcionar una configuración durante la inicialización del SDK:

```java
package hello.world;

import java.lang.Exception;

```

<div id="error-handling">
  ## Manejo de errores
</div>

El manejo de errores en este SDK debería ajustarse en gran medida a lo esperado. Todas las operaciones devuelven un objeto de respuesta o generan una excepción.

De forma predeterminada, un error de la API lanza una excepción `models/errors/APIException`. Cuando se especifican respuestas de error personalizadas para una operación, el SDK también puede lanzar la excepción asociada. Puede consultar las tablas de *Errores* correspondientes en la documentación del SDK para obtener más información sobre los posibles tipos de excepción de cada operación. Por ejemplo, el método `get` lanza las siguientes excepciones:

| Tipo de error                  | Código de estado | Tipo de contenido |
| ------------------------------ | ---------------- | ----------------- |
| models/errors/ApiErrorResponse | 400, 500         | application/json  |
| models/errors/APIException     | 4XX, 5XX         | */*               |

### Ejemplo

```java
package hello.world;

import java.lang.Exception;
import org.benzinga.BZClient.Bzclient;
import org.benzinga.BZClient.models.errors.ApiErrorResponse;
import org.benzinga.BZClient.models.operations.GetAnalystInsightsV1Request;
import org.benzinga.BZClient.models.operations.GetAnalystInsightsV1Response;

public class Application {

    public static void main(String[] args) throws ApiErrorResponse, Exception {

        Bzclient sdk = Bzclient.builder()
                .apiKeyAuth("<YOUR_API_KEY_HERE>")
            .build();

        GetAnalystInsightsV1Request req = GetAnalystInsightsV1Request.builder()
                .build();

        GetAnalystInsightsV1Response res = sdk.analystInsights().get()
                .request(req)
                .call();

        if (res.modelsAnalystInsightsJSON().isPresent()) {
            // manejar respuesta
        }
    }
}
```

<div id="select-server-by-index">
  ### Seleccione el servidor por índice
</div>

Puede sobrescribir globalmente el servidor predeterminado mediante el método del builder `.serverIndex(int serverIdx)` al inicializar la instancia del cliente SDK. El servidor seleccionado se usará luego como predeterminado en las operaciones que lo utilicen. Esta tabla enumera los índices asociados a los servidores disponibles:

| # | Servidor                            |
| - | ----------------------------------- |
| 0 | `https://api.benzinga.com`          |
| 1 | `https://api.benzinga.com/api/v1`   |
| 2 | `https://api.benzinga.com/api/v2`   |
| 3 | `https://api.benzinga.com/api/v2.1` |
| 4 | `https://api.benzinga.com/api/v2.2` |

<div id="maturity">
  ## Madurez
</div>

Este SDK está en beta y puede haber cambios incompatibles entre versiones sin que haya una actualización de la versión principal. Por lo tanto, recomendamos fijarlo
a una versión específica del paquete. De este modo, podrá instalar siempre la misma versión sin cambios incompatibles, salvo que esté buscando intencionalmente
la versión más reciente.
