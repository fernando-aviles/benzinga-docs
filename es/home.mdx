Aquí, en el portal de documentación para desarrolladores de [Benzinga Cloud](https://cloud.benzinga.com/), encontrará documentación sobre nuestras API de noticias, datos de mercado y datos de empresas. Si tiene alguna pregunta, [contáctenos](mailto:partners@benzinga.com). Estaremos encantados de ayudarle.

<div id="about-this-documentation-and-the-new-features">
  ## Acerca de esta documentación y de las nuevas funciones
</div>

Para nuestras API REST, esta documentación se basa en especificaciones OpenAPI. Puede ver el archivo YAML sin procesar en el repositorio público de esta aplicación de documentación, [aquí](https://github.com/Benzinga/doc-site-mintlify).

Hay un cliente REST integrado directamente en la sección &quot;Playground&quot; de cada página basada en OpenAPI. Una vez que haya obtenido una clave API, puede probar solicitudes sin salir del portal de documentación. Como alternativa, puede probar solicitudes con cualquier otro cliente REST con interfaz gráfica ([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/)) o con herramientas de línea de comandos como cUrl1.

Por último, las especificaciones OpenAPI son compatibles con herramientas del ecosistema Swagger, por lo que incluso puede [generar un cliente para el lenguaje de programación que elija](http://swagger.io/swagger-codegen/).

<Info>
  Nuestra documentación incorpora funciones de IA. Pruebe a hacer una pregunta con Ctrl + K (PC) o Cmd + K (Mac).
</Info>

<div id="best-practices">
  ## Mejores prácticas
</div>

Sugerimos que los clientes usen, cuando sea posible, los clientes proporcionados por Benzinga. Muchos puntos finales están disponibles en nuestra [biblioteca de Python](https://github.com/Benzinga/benzinga-python-client). Además, ofrecemos implementaciones de ejemplo y bibliotecas cliente a través de nuestro [GitHub](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente usar deltas al utilizar las API de Calendar, Signals y News, especialmente para la ingesta en tiempo real (en lugar de la ingesta de datos históricos). Esto puede hacerse mediante el parámetro de consulta `parameters[updated]` para las API de Calendar y Signals, y el parámetro de consulta `updatedSince` para la API de News. El uso de estos parámetros garantiza una latencia mínima.
