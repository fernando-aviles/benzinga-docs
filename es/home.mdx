En el portal de documentación para desarrolladores de [Benzinga Cloud](https://cloud.benzinga.com/), encontrarás documentación sobre nuestras API de noticias, mercado y datos de empresas. Si tienes alguna pregunta, [ponte en contacto con nosotros](mailto:partners@benzinga.com). Estaremos encantados de ayudarte.

<div id="about-this-documentation">
  ## Acerca de esta documentación
</div>

En el caso de nuestras API REST, esta documentación se basa en especificaciones OpenAPI. Puede ver la especificación YAML sin procesar en el repositorio público de esta aplicación de documentación, [aquí](https://github.com/Benzinga/doc-site-mintlify).

Hay un cliente REST integrado directamente en la sección &quot;Playground&quot; de cada página basada en OpenAPI. Una vez que haya obtenido una clave de API, puede probar llamadas sin salir del portal de documentación. También puede probar llamadas con cualquier otro cliente REST con interfaz gráfica ([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/)) o herramientas de CLI como cUrl1.

Por último, las especificaciones OpenAPI son compatibles con herramientas del ecosistema Swagger, por lo que incluso puede [generar un cliente para el lenguaje de programación que prefiera](http://swagger.io/swagger-codegen/).

<Info>
  Nuestra documentación cuenta con funciones de IA. Pruebe a hacer una pregunta con Ctrl + K (PC) o Cmd + K (Mac).
</Info>

<div id="best-practices">
  ## Mejores prácticas
</div>

Sugerimos que los clientes utilicen los clientes de Benzinga siempre que sea posible. Muchos endpoints están disponibles en nuestra [biblioteca de Python](https://github.com/Benzinga/benzinga-python-client). También ofrecemos implementaciones de ejemplo y bibliotecas cliente a través de nuestro [Github](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente usar deltas al utilizar las API de Calendar, Signals y Noticias, especialmente para la ingesta en tiempo real (en lugar de la ingesta de datos históricos). Esto puede hacerse mediante el parámetro de consulta `parameters[updated]` para las API de Calendar y Signals, y el parámetro de consulta `updatedSince` para la API de Noticias. El uso de estos parámetros garantiza una latencia mínima.
