Aquí, en el portal de documentación para desarrolladores de [Benzinga Cloud](https://cloud.benzinga.com/), encontrarás documentación sobre nuestras APIs de Noticias, mercado y datos de company. Si tienes alguna pregunta, [contáctanos](mailto:partners@benzinga.com). Estaremos encantados de ayudarte.

<div id="about-this-documentation">
  ## Acerca de esta documentación
</div>

Para nuestras APIs REST, esta documentación se basa en especificaciones OpenAPI. Puede ver la especificación YAML sin procesar consultando el repositorio público de esta aplicación de documentación, [aquí](https://github.com/Benzinga/doc-site-mintlify).

Hay un cliente REST integrado directamente en la sección &quot;Playground&quot; de cada página basada en OpenAPI. Una vez que haya obtenido una clave de API, puede probar solicitudes sin salir del portal de documentación. Como alternativa, puede probar solicitudes con cualquier otro cliente REST con interfaz gráfica ([Postman](https://www.getpostman.com/), [Insomnia](https://insomnia.rest/), [Paw](https://paw.cloud/)) o con herramientas de línea de comandos como cUrl1.

Por último, las especificaciones OpenAPI son compatibles con las herramientas del ecosistema Swagger, por lo que incluso puede [generar un cliente para el lenguaje de programación que prefiera](http://swagger.io/swagger-codegen/).

</div>

</Steps>

<div id="best-practices">
  ## Mejores prácticas
</div>

Sugerimos que los clientes usen, siempre que sea posible, los clientes de Benzinga. Muchos endpoints están disponibles en nuestra [biblioteca de Python](https://github.com/Benzinga/benzinga-python-client). Además, ofrecemos implementaciones de ejemplo y bibliotecas cliente a través de nuestro [Github](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente emplear deltas al utilizar las API de Calendario, Señales y Noticias, especialmente cuando se usen para la ingesta en tiempo real (en lugar de la ingesta de datos históricos). Esto puede hacerse mediante el parámetro de consulta `parameters[updated]` para las API de Calendario y Señales, y el parámetro de consulta `updatedSince` para la API de Noticias. El uso de estos parámetros garantiza una latencia mínima.
