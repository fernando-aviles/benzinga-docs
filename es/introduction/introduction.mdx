<div id="benzinga-apis">
  ## API de Benzinga
</div>

Desde informes de resultados hasta actividad de opciones, Benzinga se centra en la información que mueve el mercado. Ofrecemos un conjunto de canales centrado en noticias de última hora para que su empresa pueda ser la primera en cerrar una operación. Ofrecemos dos formas de recibir nuestras noticias: mediante una API REST basada en consultas o mediante TCP basado en envío. Ambas ofrecen un acceso sin precedentes a los mercados.

Uno de los principios fundamentales de Benzinga es &quot;obsesionarnos con nuestros clientes, no con nuestra competencia&quot;.

Las relaciones sólidas son la fuerza impulsora de los negocios exitosos, y nuestro compromiso con ello es siempre lo primero.

Para empezar a usar la API de Benzinga, envíenos un correo electrónico directamente a licensing@benzinga.com o llámenos al 877-440-9464. Responderemos en el plazo de un día hábil.

<div id="authentication">
  ## Autenticación
</div>

Incluya su clave o token de API como el parámetro token en la cadena de consulta de la solicitud.

<div id="best-practices">
  ## Mejores prácticas
</div>

Sugerimos que los clientes usen, siempre que sea posible, los clientes de Benzinga. Muchos puntos finales están disponibles en nuestra [biblioteca de Python](/es/python-client). Proporcionamos implementaciones de referencia y bibliotecas cliente en nuestro [Github](https://github.com/benzinga) público.

Por último, recomendamos encarecidamente usar deltas al utilizar las API de Calendar, Signals y News, especialmente para la ingesta en tiempo real (en lugar de la ingesta de datos históricos). Esto puede hacerse mediante el parámetro de consulta `parameters[updated]` para las API de Calendar y Signals, y el parámetro de consulta `updatedSince` para la API de News. El uso de estos parámetros garantiza una latencia mínima.

<div id="max-offset">
  ## Offset máximo
</div>

La paginación de las API de News, Calendar y Signals no puede devolver conjuntos de resultados con más de 10.000 elementos. Es decir, si el tamaño de página es 1.000, no puedes solicitar la página 10.

Para sortear esta limitación, debes ajustar el alcance de la consulta. Por ejemplo, puedes restringirla a un intervalo de fechas o a un conjunto específico de tíkers que incluya menos de 10.000 elementos. Después, puedes ajustar la consulta para incluir otro intervalo de fechas o conjunto de tíkers que incluya menos de 10.000 elementos, combinar los conjuntos y repetir el proceso hasta obtener el conjunto completo de resultados que buscas.

<div id="return-format">
  ## Formato de respuesta
</div>

El formato predeterminado de las API de Benzinga se especifica en las especificaciones individuales de cada API, pero será `JSON` salvo que se indique lo contrario.

Puede solicitar `JSON` configurando el encabezado `accept` como `application/json`, o `XML` con `application/xml` cuando no esté en desuso.

**No se recomiendan las nuevas integraciones que utilicen el formato de salida `XML`.**

<div id="example">
  ### Ejemplo
</div>

```JSON
curl -X GET \
'http://api.benzinga.com/api/v2/calendar/ratings?token=YOUR_TOKEN_HERE&ratings=Upgrades' \
-H 'accept: application/json'
```
